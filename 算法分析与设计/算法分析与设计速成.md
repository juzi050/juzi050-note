## 第 1 章 算法在计算中的作用

### 算法基本概念

| 概念 | 说明 |
| --- | --- |
| **问题** | 抽象描述 |
| **实例** | 问题的具体化（如给定多项式具体参数后求值） |
| **算法** | 求解某个问题的**长度有限的指令序列**，每条指令都是确定的、简单的、机械的和可执行的 |

### 算法的性质

- **正确性**：对每一个输入实例，都能输出正确结果并停止 → 正确的算法；不正确的算法可能不停止或结果错误（但不一定没用，可用于近似/模拟）
- **可计算性**：理论上判断什么问题可以给出算法求解。"停机问题"不可计算
- **复杂性**：时间复杂性和空间复杂性。本课程着重讨论**时间复杂性**，从数学角度分析基本操作的次数与问题规模的关系

### 算法效率示例：多项式求值

两种算法对比 $p(x) = a_n x^n + a_{n-1} x^{n-1} + \dots + a_1 x + a_0$：

| 算法 | 乘法次数 | 加法次数 |
| --- | --- | --- |
| 算法1（逐项计算） | $2n$ | $n$ |
| 算法2（**秦九韶/Horner**） | $n$ | $n$ |

**算法1（逐项计算）伪码**：
```
1. p ← a_0
2. y ← 1
3. 对 i 从 1 到 n 做
4.     y ← y·x
5.     p ← p + a_i·y
6. 输出 p
```

**算法1 C 语言**：
```c
/* 逐项计算: a[0] 是常数项 a_0, a[1] 是一次项系数... */
int poly_naive(int a[], int n, int x) {
    int p = a[0], y = 1;
    for (int i = 1; i <= n; i++) {
        y = y * x;          /* 计算 x^i */
        p = p + a[i] * y;
    }
    return p;
}
```

**算法2（秦九韶/Horner）伪码**：
```
1. p ← a_n
2. 对 i 从 n 到 1 做
3.     p ← p·x + a_{i-1}
4. 输出 p
```

**算法2 C 语言**：
```c
/* Horner 方法: a[0] 是最高次系数 a_n */
int horner(int a[], int n, int x) {
    int p = a[0];
    for (int i = 1; i <= n; i++)
        p = p * x + a[i];
    return p;
}
```

## 第 2 章 算法入门

### 2.1 插入排序（Insertion Sort）

**思想**：像打牌一样，摸起一张牌插入到左手已排好序的正确位置。

**伪码**：
```
INSERTION-SORT(A)
1  for j ← 2 to length[A]
2      do key ← A[j]
3          i ← j - 1
4          while i > 0 and A[i] > key
5              do A[i+1] ← A[i]
6                  i ← i - 1
7          A[i+1] ← key
```

**C 语言**：
```c
void insertion_sort(int A[], int n) {
    int i, j, key;
    for (j = 1; j < n; j++) {       /* 伪码 j:2→n 对应 C:1→n-1 */
        key = A[j];
        i = j - 1;
        while (i >= 0 && A[i] > key) {
            A[i + 1] = A[i];
            i--;
        }
        A[i + 1] = key;
    }
}
```

**复杂度**：
- 最好情况（已排序）：$T(n) = an + b = \Theta(n)$
- 最坏情况（逆序）：$T(n) = \Theta(n^2)$
- 平均情况：$\Theta(n^2)$

### 2.2 循环不变式

用于证明算法正确性，类比数学归纳法：

| 性质 | 说明 |
| --- | --- |
| **初始化** | 第一轮迭代开始前，不变式正确 |
| **保持** | 若某次迭代开始前正确，则下次迭代开始前仍正确 |
| **终止** | 循环结束时，不变式给出有用性质，表明算法正确 |

**插入排序的循环不变式**：在外层 for 循环每一轮开始，子数组 $A[1..j-1]$ 已排好序。

### 2.3 RAM 模型

- 通用的单处理器、随机存取 RAM 计算模型
- 指令一条接一条执行，无并发
- 每条指令时间为常量
- 包含算术指令、数据移动指令、控制指令
- 对真实计算机有良好预测

### 2.4 分治法（Divide and Conquer）

**三个步骤**：
1. **分解（Divide）**：将原问题分解成若干子问题
2. **解决（Conquer）**：递归地解各子问题，足够小时直接求解
3. **合并（Combine）**：将子问题的结果合并

#### 合并排序（Merge Sort）

**伪码**：
```
MERGE-SORT(A, p, r)
1  if p < r
2      then q ← ⌊(p + r)/2⌋
3           MERGE-SORT(A, p, q)
4           MERGE-SORT(A, q+1, r)
5           MERGE(A, p, q, r)
```

**MERGE 伪码**（时间 $\Theta(n)$，哨兵 $\infty$ 简化代码）：
```
MERGE(A, p, q, r)
1  n1 ← q - p + 1;   n2 ← r - q
2  create L[1..n1+1], R[1..n2+1]
3  for i ← 1 to n1 do L[i] ← A[p + i - 1]
4  for j ← 1 to n2 do R[j] ← A[q + j]
5  L[n1 + 1] ← ∞;   R[n2 + 1] ← ∞
6  i ← 1;  j ← 1
7  for k ← p to r
8      if L[i] ≤ R[j] then A[k] ← L[i]; i++
9      else A[k] ← R[j]; j++
```

**C 语言**：
```c
void merge(int A[], int p, int q, int r) {
    int n1 = q - p + 1, n2 = r - q;
    int L[n1 + 1], R[n2 + 1];
    int i, j, k;
    for (i = 0; i < n1; i++) L[i] = A[p + i];
    for (j = 0; j < n2; j++) R[j] = A[q + 1 + j];
    L[n1] = R[n2] = 2147483647;  /* 哨兵: INT_MAX */
    i = j = 0;
    for (k = p; k <= r; k++) {
        if (L[i] <= R[j]) A[k] = L[i++];
        else              A[k] = R[j++];
    }
}

void merge_sort(int A[], int p, int r) {
    if (p < r) {
        int q = (p + r) / 2;
        merge_sort(A, p, q);
        merge_sort(A, q + 1, r);
        merge(A, p, q, r);
    }
}
```

**复杂度分析**：
$$T(n) = \begin{cases} \Theta(1) & n = 1 \\ 2T(n/2) + \Theta(n) & n > 1 \end{cases}$$

由主定理得：$T(n) = \Theta(n \log n)$

**递归树**：树高 $\log n$，每层代价 $cn$，叶子 $n$ 个 → 总代价 $\Theta(n \log n)$。

**与插入排序对比**：
- 当输入规模足够大时，合并排序 $\Theta(n \log n)$ 在最坏情况下比插入排序 $\Theta(n^2)$ 好
- 对数函数增长速度比任何线性函数都要慢

### 伪代码约定

1. 缩进表示分程序结构
2. 循环/条件与 Pascal 类似；退出循环后计数器保持
3. `▹` 表示注释
4. 多重赋值 `i ← j ← e`
5. 变量局部于给定过程
6. 数组通过 `数组名[下标]` 访问，`..` 表示范围
7. 复合数据组织成对象，有属性/域
8. 参数按值传递（对象传指针）
9. `and` 和 `or` 有短路能力

## 第 3 章 函数的增长

### 3.1 渐近记号

| 记号 | 含义 | 数学定义 | 类比实数 |
| --- | --- | --- | --- |
| $\Theta(g(n))$ | **渐近确界** | $0 \le c_1 g(n) \le f(n) \le c_2 g(n)$ | $a = b$ |
| $O(g(n))$ | **渐近上界** | $0 \le f(n) \le cg(n)$ | $a \le b$ |
| $\Omega(g(n))$ | **渐近下界** | $0 \le cg(n) \le f(n)$ | $a \ge b$ |

**注意**：$\Theta(g(n)) \subseteq O(g(n))$ 且 $\Theta(g(n)) \subseteq \Omega(g(n))$。

**性质**：
- **传递性**：三个记号都具有
- **自反性**：$f(n) = \Theta(f(n))$ 等
- **对称性**：$f(n) = \Theta(g(n)) \iff g(n) = \Theta(f(n))$
- **转置对称性**：$f(n) = O(g(n)) \iff g(n) = \Omega(f(n))$
- **三分性不成立**：两个函数可能无法用渐近记号比较（如 $n$ 和 $n^{1+\sin n}$）

**证明示例**：
- $\frac{1}{2}n^2 - 3n = \Theta(n^2)$：取 $c_1 = 1/14, c_2 = 1/2, n_0 = 7$
- $6n^3 \neq \Theta(n^2)$：反证法

### 3.2 标准记号和常用函数

| 函数/概念 | 关键性质 |
| --- | --- |
| **单调性** | 单调递增/递减；严格递增/递减 |
| **下取整 $\lfloor x \rfloor$** | $\lfloor n/2 \rfloor + \lceil n/2 \rceil = n$ |
| **上取整 $\lceil x \rceil$** | $x-1 < \lfloor x \rfloor \le x \le \lceil x \rceil < x+1$ |
| **取模 $a \bmod n$** | $a - \lfloor a/n \rfloor \cdot n$；$a \equiv b \pmod{n}$ |
| **多项式** | $p(n) = \sum a_i n^i$，若 $a_d > 0$ 则 $p(n) = \Theta(n^d)$ |
| **指数式** | $n^b = o(a^n)$，即多项式比指数增长慢 |
| **对数** | $\log n = \log_2 n$；$\ln n = \log_e n$；$n^b = o(a^n)$ |
| **阶乘** | $n! = o(n^n)$；$n! = \omega(2^n)$；$\log(n!) = \Theta(n \log n)$ |
| **多重对数** | $\log^* n = \min\{i \ge 0 : \log^{(i)} n \le 1\}$，增长极慢 |
| **斐波那契数** | $F_i = F_{i-1} + F_{i-2}$，以指数形式增长 |

## 第 4 章 递归式

### 三种解法

| 方法 | 描述 |
| --- | --- |
| **代换法** | 猜测解的形式 → 数学归纳法证明 |
| **递归树** | 树形结构 → 对和式限界 |
| **主方法** | $T(n) = aT(n/b) + f(n)$ 的"食谱"解法 |

### 4.1 代换法

**步骤**：
1. 猜测解的形式（如 $O(n \log n)$）
2. 用数学归纳法找出使解有效的常数
3. 注意边界条件（$n=1$ 可能不满足，取更大的 $n_0$）

**技巧**：
- 减掉一个低阶项：当 $T(n) \le cn + 1$ 不满足时，尝试 $T(n) \le cn - b$
- 改变变量：如 $m = \log n$ 简化递归式
- 避免陷阱：证明 $T(n) = O(n)$ 不等价于 $T(n) \le cn + n = O(n)$（必须证出 $\le cn$）

### 4.2 递归树方法

- 每个节点代表递归调用的代价
- 将各层代价求和
- 树深取决于子问题规模缩小的速度
- 通常能容忍小量"不良量"

**例**：$T(n) = 3T(n/4) + \Theta(n^2)$
- 树深：$\log_4 n$
- 根代价：$cn^2$，每层按 $(3/16)^i$ 递减
- 总代价：$T(n) = \Theta(n^2)$

### 4.3 主方法（Master Theorem）

**形式**：$T(n) = aT(n/b) + f(n)$，其中 $a \ge 1$，$b > 1$

**主方法用法**——比较 $n^{\log_b a}$ 与 $f(n)$：

| **若** $n^{\log_b a} > f(n)$ | 则 $T(n) = \Theta(n^{\log_b a})$                              |
| --------------------------- | ------------------------------------------------------------ |
| **若** $n^{\log_b a} = f(n)$ | 则 $T(n) = \Theta(n^{\log_b a} \log n) = \Theta(f(n) \log n)$ |
| **若** $n^{\log_b a} < f(n)$ | 则 $T(n) = \Theta(f(n))$                                      |

**应用示例**：

| 递归式                        | $a$ | $b$ | $n^{\log_b a}$ | 比较                       | 解                 |
| -------------------------- | --- | --- | -------------- | ------------------------ | ----------------- |
| $T(n) = 9T(n/3) + n$       | 9   | 3   | $n^2$          | $n^2 > n$                | $\Theta(n^2)$     |
| $T(n) = T(2n/3) + 1$       | 1   | 3/2 | $1$            | $1 = 1$                  | $\Theta(\log n)$   |
| $T(n) = 3T(n/4) + n \log n$ | 3   | 4   | $n^{\log_4 3}$ | $n^{\log_4 3} < n \log n$ | $\Theta(n \log n)$ |

**关键注解**：
- 上述简单比较是直觉版本。严格来说要求"**多项式地**"大于/小于（差 $n^\varepsilon$ 因子），否则无法用。例如 $T(n) = 2T(n/2) + n \log n$ 中 $n \log n$ 不是多项式大于 $n$，主方法失效。
- 情况 $n^{\log_b a} < f(n)$ 还需**规则性条件** $af(n/b) \le cf(n)$（$c < 1$）。

## 第 6 章 堆排序

### 6.1 堆的定义

- **（二叉）堆**：一种数组对象，可视为**完全二叉树**
- `length[A]`：数组元素个数；`heap-size[A]`：堆中元素个数（$0 \le$ heap-size $\le$ length）
- 下标关系（1-based）：

| 关系 | 公式 |
| --- | --- |
| 父节点 PARENT(i) | $\lfloor i/2 \rfloor$ |
| 左儿子 LEFT(i) | $2i$ |
| 右儿子 RIGHT(i) | $2i+1$ |

- **最大堆**：$A[\text{PARENT}(i)] \ge A[i]$，根最大
- **最小堆**：$A[\text{PARENT}(i)] \le A[i]$，根最小
- 堆的高度：$\Theta(\log n)$

### 6.2 保持堆的性质 — MAX-HEAPIFY

**功能**：调整 $A[i]$ 位置，使以 $i$ 为根的子树成为最大堆

**伪码**：
```
MAX-HEAPIFY(A, i)
1  l ← LEFT(i)
2  r ← RIGHT(i)
3  if l ≤ heap-size[A] and A[l] > A[i]    then largest ← l
4  else largest ← i
5  if r ≤ heap-size[A] and A[r] > A[largest] then largest ← r
6  if largest ≠ i
7      then exchange A[i] ↔ A[largest]
8           MAX-HEAPIFY(A, largest)
```

**C 语言**（堆用 1-based 数组，A[0] 闲置或存 heap_size）：
```c
/* 堆用 1-based 数组: A[1..n], A[0] 存 heap_size */
void max_heapify(int A[], int i) {
    int l = 2 * i, r = 2 * i + 1, largest = i;
    int size = A[0];   /* A[0] 存放当前 heap_size */
    if (l <= size && A[l] > A[largest]) largest = l;
    if (r <= size && A[r] > A[largest]) largest = r;
    if (largest != i) {
        int tmp = A[i]; A[i] = A[largest]; A[largest] = tmp;
        max_heapify(A, largest);
    }
}
```

**时间复杂度**：$T(n) \le T(2n/3) + \Theta(1) \implies T(n) = O(\log n)$

### 6.3 建堆 — BUILD-MAX-HEAP

**自底向上**调用 MAX-HEAPIFY：

```
BUILD-MAX-HEAP(A)
1  heap-size[A] ← length[A]
2  for i ← ⌊length[A]/2⌋ downto 1
3      do MAX-HEAPIFY(A, i)
```

**C 语言**：
```c
void build_max_heap(int A[], int n) {
    A[0] = n;   /* 用 A[0] 存 heap_size */
    for (int i = n / 2; i >= 1; i--)
        max_heapify(A, i);
}
```

**精确上界**：$O(n)$（不是简单的 $O(n \log n)$，因为多数节点高度很小）

### 6.4 堆排序 — HEAPSORT

```
HEAPSORT(A)
1  BUILD-MAX-HEAP(A)
2  for i ← length[A] downto 2
3      do exchange A[1] ↔ A[i]
4          heap-size[A] ← heap-size[A] - 1
5          MAX-HEAPIFY(A, 1)
```

**C 语言**：
```c
void heapsort(int A[], int n) {
    build_max_heap(A, n);   /* A[0]=n, 堆建立在 A[1..n] */
    for (int i = n; i >= 2; i--) {
        int tmp = A[1]; A[1] = A[i]; A[i] = tmp;  /* swap */
        A[0]--;              /* heap_size-- */
        max_heapify(A, 1);
    }
}
```

**时间复杂度**：$O(n \log n)$（建堆 $O(n)$ + $n$ 次 MAX-HEAPIFY 各 $O(\log n)$）

### 6.5 优先级队列

最大优先级队列操作：

| 操作 | 功能 | 时间 |
| --- | --- | --- |
| MAXIMUM(S) | 返回最大关键字元素 | $\Theta(1)$ |
| EXTRACT-MAX(S) | 去掉并返回最大元素 | $O(\log n)$ |
| INCREASE-KEY(S, x, k) | 将 $x$ 的关键字增加到 $k$ | $O(\log n)$ |
| INSERT(S, x) | 插入元素 $x$ | $O(\log n)$ |

**HEAP-INCREASE-KEY**：自底向上与父节点比较交换
**MAX-HEAP-INSERT**：在末尾插入 $-\infty$，再调用 HEAP-INCREASE-KEY

**C 语言**（4 个优先级队列操作）：
```c
int heap_maximum(int A[]) { return A[1]; }

int heap_extract_max(int A[]) {
    if (A[0] < 1) return -1;   /* heap underflow */
    int max = A[1];
    A[1] = A[A[0]];  A[0]--;   /* 末尾元素移到根，size-- */
    max_heapify(A, 1);
    return max;
}

void heap_increase_key(int A[], int i, int key) {
    if (key < A[i]) return;     /* 新 key 不能比原值小 */
    A[i] = key;
    while (i > 1 && A[i / 2] < A[i]) {  /* 与父节点比较 */
        int tmp = A[i]; A[i] = A[i/2]; A[i/2] = tmp;
        i = i / 2;
    }
}

void max_heap_insert(int A[], int key) {
    A[0]++;                     /* heap_size++ */
    A[A[0]] = -2147483648;      /* 哨兵: INT_MIN */
    heap_increase_key(A, A[0], key);
}
```

## 第 7 章 快速排序

### 7.1 算法描述

**三步骤**：
1. **分解**：选主元，划分 $A[p..q-1] \le A[q] < A[q+1..r]$
2. **解决**：递归排序两个子数组
3. **合并**：就地排序，无需额外操作

**伪码**：
```
QUICKSORT(A, p, r)
1  if p < r
2      then q ← PARTITION(A, p, r)
3           QUICKSORT(A, p, q-1)
4           QUICKSORT(A, q+1, r)
```

**PARTITION 伪码**（时间复杂度 $\Theta(n)$）：
```
PARTITION(A, p, r)
1  x ← A[r]
2  i ← p - 1
3  for j ← p to r - 1
4      do if A[j] ≤ x
5          then i ← i + 1
6               exchange A[i] ↔ A[j]
7  exchange A[i+1] ↔ A[r]
8  return i + 1
```

**C 语言**：
```c
int partition(int A[], int p, int r) {
    int x = A[r];                  /* 主元 */
    int i = p - 1;
    for (int j = p; j < r; j++) {
        if (A[j] <= x) {
            i++;
            int tmp = A[i]; A[i] = A[j]; A[j] = tmp;
        }
    }
    int tmp = A[i + 1]; A[i + 1] = A[r]; A[r] = tmp;
    return i + 1;
}

void quicksort(int A[], int p, int r) {
    if (p < r) {
        int q = partition(A, p, r);
        quicksort(A, p, q - 1);
        quicksort(A, q + 1, r);
    }
}
```

**循环不变式**：
- 若 $p \le k \le i$，则 $A[k] \le x$
- 若 $i+1 \le k \le j-1$，则 $A[k] > x$
- 若 $k = r$，则 $A[k] = x$

### 7.2 性能分析

| 情况 | 递归式 | 时间复杂度 |
| --- | --- | --- |
| **最坏情况**（划分 0 : n-1） | $T(n) = T(n-1) + \Theta(n)$ | $\Theta(n^2)$ |
| **最佳情况**（划分 n/2 : n/2） | $T(n) = 2T(n/2) + \Theta(n)$ | $\Theta(n \log n)$ |
| **平衡划分**（如 9 : 1） | $T(n) = T(9n/10) + T(n/10) + cn$ | $\Theta(n \log n)$ |

**关键结论**：
- 只要划分是常数比例的，运行时间都是 $\Theta(n \log n)$
- 好坏划分交替出现时，仍是 $\Theta(n \log n)$（常数因子略大）

### 7.3 随机化快速排序

**动机**：避免对特定输入产生最坏情况。

**方法**：随机选择主元，将 $A[r]$ 与随机元素交换。

```
RANDOMIZED-PARTITION(A, p, r)
1  i ← RANDOM(p, r)
2  exchange A[r] ↔ A[i]
3  return PARTITION(A, p, r)
```

**C 语言**（用 `rand()` 模拟随机数）：
```c
#include <stdlib.h>
int randomized_partition(int A[], int p, int r) {
    int i = p + rand() % (r - p + 1);  /* 随机选 p..r 间一个下标 */
    int tmp = A[r]; A[r] = A[i]; A[i] = tmp;
    return partition(A, p, r);
}

void randomized_quicksort(int A[], int p, int r) {
    if (p < r) {
        int q = randomized_partition(A, p, r);
        randomized_quicksort(A, p, q - 1);
        randomized_quicksort(A, q + 1, r);
    }
}
```

**平均情况期望时间**：$\Theta(n \log n)$

### 7.4 快速排序分析

- **最坏情况证明**：$T(n) = \max_{0 \le q \le n-1}(T(q) + T(n-q-1)) + \Theta(n)$，代入 $T(n) \le cn^2$ 可证
- **平均情况**：指示器随机变量 $X_{ij}$ 表示 $z_i$ 与 $z_j$ 是否比较，期望比较次数 $E[X] = \sum_{i=1}^{n-1} \sum_{j=i+1}^{n} \frac{2}{j-i+1} = O(n \log n)$

## 第 8 章 线性时间排序

### 8.1 比较排序的下界

- **决策树模型**：比较排序可抽象为决策树（满二叉树）
- 每个叶节点对应一种排列，$n$ 个元素共有 $n!$ 种排列
- 树高 $h \ge \log(n!) = \Omega(n \log n)$

> **定理 8.1**：任意比较排序算法在最坏情况下都需要 $\Omega(n \log n)$ 次比较。

**推论**：堆排序和归并排序是**渐近最优**的比较排序算法。

### 8.2 计数排序（Counting Sort）

**前提**：$n$ 个输入元素是 $0$ 到 $k$ 之间的整数，$k = O(n)$

**思想**：确定小于每个元素 $x$ 的元素个数，直接将 $x$ 放到输出数组的对应位置。

**伪码**：
```
COUNTING-SORT(A, B, k)
1  for i ← 0 to k do C[i] ← 0
2  for j ← 1 to length[A] do C[A[j]] ← C[A[j]] + 1
3  ▹ C[i] 现在包含等于 i 的元素个数
4  for i ← 1 to k do C[i] ← C[i] + C[i-1]
5  ▹ C[i] 现在包含小于或等于 i 的元素个数
6  for j ← length[A] downto 1
7      do B[C[A[j]]] ← A[j]
8          C[A[j]] ← C[A[j]] - 1
```

**C 语言**：
```c
void counting_sort(int A[], int B[], int n, int k) {
    int C[k + 1];                  /* n = length(A), k 是值域上限 */
    int i;
    for (i = 0; i <= k; i++) C[i] = 0;
    for (i = 0; i < n;  i++) C[A[i]]++;
    for (i = 1; i <= k; i++) C[i] += C[i - 1];
    for (i = n - 1; i >= 0; i--) { /* downto 保证稳定性 */
        B[C[A[i]] - 1] = A[i];     /* C 是 1-based 计数, B 0-based */
        C[A[i]]--;
    }
}
```

**时间复杂度**：$\Theta(k+n)$；当 $k = O(n)$ 时，为 $O(n)$

**特性**：**稳定排序**（`downto` 保证）。不是基于比较的排序算法。

### 8.3 基数排序（Radix Sort）

**思想**：按最低有效位优先排序。

```
RADIX-SORT(A, d)
1  for i ← 1 to d
2      do use a stable sort to sort array A on digit i
```

**C 语言**（以 10 进制为例，每轮用计数排序）：
```c
/* 提取 key 的第 dig 位 (dig=0为个位) */
int digit(int key, int dig) {
    while (dig--) key /= 10;
    return key % 10;
}

void radix_sort(int A[], int n, int d) {  /* d 为最大位数 */
    int B[n];
    for (int dig = 0; dig < d; dig++) {
        int C[10] = {0};                  /* 每位 0~9 */
        for (int i = 0; i < n;  i++) C[digit(A[i], dig)]++;
        for (int i = 1; i < 10; i++) C[i] += C[i - 1];
        for (int i = n - 1; i >= 0; i--) {
            int dg = digit(A[i], dig);
            B[--C[dg]] = A[i];            /* 注意 C 预减 */
        }
        for (int i = 0; i < n; i++) A[i] = B[i];  /* 复制回 A */
    }
}
```

**引理 8.3**：$n$ 个 $d$ 位数，每位 $k$ 种值 → 时间 $\Theta(d(n+k))$

**引理 8.4**：$n$ 个 $b$ 位数，取 $r \le b$ 位一组 → 时间 $\Theta((b/r)(n+2^r))$

**与比较排序的对比**：
- 虽然基数排序 $O(n)$ 快于快速排序 $O(n \log n)$，但常数因子不同
- 基数排序每遍时间可能更长，且非原地排序

### 8.4 桶排序（Bucket Sort）

**前提**：输入均匀分布在 $[0, 1)$ 上

**思想**：将 $[0, 1)$ 划分为 $n$ 个等大小的桶，分布元素后各桶内插入排序，再按序合并。

**C 语言**（简化版：用二维数组模拟桶链表，每个桶内插排）：
```c
void bucket_sort(float A[], int n) {  /* 假设 A[i] ∈ [0, 1) */
    /* 桶: bucket[i][0] 存该桶元素个数, bucket[i][1..] 存元素 */
    float bucket[n][n + 1];
    int i, j, k, idx = 0;
    for (i = 0; i < n; i++) bucket[i][0] = 0;  /* 桶空 */
    /* 分配: floor(n * A[i]) 决定桶 */
    for (i = 0; i < n; i++) {
        int b = (int)(n * A[i]);
        bucket[b][0] = bucket[b][0] + 1;
        bucket[b][(int)bucket[b][0]] = A[i];
    }
    /* 桶内插入排序 + 合并回 A */
    for (i = 0; i < n; i++) {
        int cnt = (int)bucket[i][0];
        /* 插入排序 */
        for (j = 2; j <= cnt; j++) {
            float key = bucket[i][j];
            k = j - 1;
            while (k >= 1 && bucket[i][k] > key) {
                bucket[i][k + 1] = bucket[i][k]; k--;
            }
            bucket[i][k + 1] = key;
        }
        for (j = 1; j <= cnt; j++) A[idx++] = bucket[i][j];
    }
}
```

**期望运行时间**：$\Theta(n)$

## 第 15 章 动态规划（Dynamic Programming）

### 核心概念

**与分治法异同**：

| | 分治法 | 动态规划 |
| --- | --- | --- |
| 相同点 | 合并子问题的解 | 合并子问题的解 |
| 不同点 | 子问题**独立** | 子问题**重叠**（共享子子问题） |
| 效率 | 可能重复计算 | 每子问题只算一次，存入表中 |

### 四个步骤

1. **描述最优解的结构特征**（最优子结构）
2. **递归地定义一个最优解的值**
3. **自底向上计算最优解的值**
4. **从已计算的信息中构造一个最优解**（可选）

### 动态规划要素

#### 最优子结构（Optimal Substructure）

一个问题的最优解包含了子问题的最优解。

- 发现方法：分析解的选择 → 确定子问题 → 用**剪切-粘贴法**反证
- 子问题空间应尽可能简单
- 运行时间 = 子问题总数 × 每个子问题中涉及的选择数

**注意**：最长路径问题**没有**最优子结构（子问题不独立，共享顶点），而最短路径有。

#### 重叠子问题（Overlapping Subproblems）

- 递归算法重复访问同一子问题
- 动态规划对每子问题只求解一次，存入表中
- 当子问题规模为多项式阶时特别有效

#### Memoization（备忘法）

- 自顶向下递归，但将解记录在表中
- 每个子问题首次遇到时计算并存储，再次遇到时直接查表
- 与自底向上 DP 相比：常数因子稍大，但只计算需要的子问题

### 实例一：装配线调度问题

**问题**：两条装配线，$n$ 个站，可在装配线间转移（有时间代价），求最短装配时间。

**递归公式**（到达 $S_{1,j}$ 的最快时间）：
$$f_1[j] = \min(f_1[j-1] + a_{1,j},\; f_2[j-1] + t_{2,j-1} + a_{1,j})$$

$$f_2[j] = \min(f_2[j-1] + a_{2,j},\; f_1[j-1] + t_{1,j-1} + a_{2,j})$$

最终答案：$f^* = \min(f_1[n] + x_1,\; f_2[n] + x_2)$

**C 语言**：
```c
/* a[2][n]: 装配时间;  t[2][n-1]: 转移时间;  e[2]:进入; x[2]:离开 */
int fastest_way(int n, int a[2][n], int t[2][n-1], int e[2], int x[2]) {
    int f1[n], f2[n];
    f1[0] = e[0] + a[0][0];
    f2[0] = e[1] + a[1][0];
    for (int j = 1; j < n; j++) {
        f1[j] = (f1[j-1] + a[0][j] < f2[j-1] + t[1][j-1] + a[0][j])
              ? (f1[j-1] + a[0][j]) : (f2[j-1] + t[1][j-1] + a[0][j]);
        f2[j] = (f2[j-1] + a[1][j] < f1[j-1] + t[0][j-1] + a[1][j])
              ? (f2[j-1] + a[1][j]) : (f1[j-1] + t[0][j-1] + a[1][j]);
    }
    return (f1[n-1] + x[0] < f2[n-1] + x[1])
         ? (f1[n-1] + x[0]) : (f2[n-1] + x[1]);
}
```

**时间复杂度**：$\Theta(n)$（自左向右迭代计算）

### 实例二：矩阵链乘（Matrix Chain Multiplication）

**问题**：$n$ 个矩阵相乘，求最小标量乘法次数的最优括号化。

**递归定义**：$m[i,j]$ 表示计算 $A_{i..j}$ 的最小乘法次数

$$m[i,j] = \begin{cases} 0 & i = j \\ \min_{i \le k < j}\{m[i,k] + m[k+1,j] + p_{i-1}p_k p_j\} & i < j \end{cases}$$

**算法**：按链长递增序计算 $m[i,j]$

**时间复杂度**：$\Theta(n^3)$（$\Theta(n^2)$ 个子问题，每个至多 $n-1$ 种选择）

**C 语言**：
```c
/* p[0..n] 存矩阵维度: A_i 维度为 p[i-1]×p[i]; m[i][j] 最优乘法次数 */
void matrix_chain_order(int p[], int n, int m[][n+1], int s[][n+1]) {
    for (int i = 1; i <= n; i++) m[i][i] = 0;   /* 链长为 1 */
    for (int L = 2; L <= n; L++) {               /* 链长递增 */
        for (int i = 1; i <= n - L + 1; i++) {
            int j = i + L - 1;
            m[i][j] = 2147483647;               /* INT_MAX */
            for (int k = i; k < j; k++) {
                int q = m[i][k] + m[k+1][j] + p[i-1]*p[k]*p[j];
                if (q < m[i][j]) { m[i][j] = q; s[i][j] = k; }
            }
        }
    }
}
```

**构造最优解**：用 $s[i,j]$ 记录分裂点 $k$，回溯构造

### 实例三：最长公共子序列（LCS）

**问题**：给定序列 $X$ 和 $Y$，求最长公共子序列。

**递归公式**（$c[i,j]$ 表示 $X_i$ 和 $Y_j$ 的 LCS 长度）：

$$c[i,j] = \begin{cases} 0 & i=0 \text{ 或 } j=0 \\ c[i-1,j-1] + 1 & x_i = y_j \\ \max(c[i-1,j], c[i,j-1]) & x_i \neq y_j \end{cases}$$

**定理 15.1**（最优子结构）：
- 若 $x_m = y_n$，则 $z_k = x_m = y_n$，且 $Z_{k-1}$ 是 $X_{m-1}$ 和 $Y_{n-1}$ 的 LCS
- 若 $x_m \neq y_n$ 且 $z_k \neq x_m$，则 $Z$ 是 $X_{m-1}$ 和 $Y$ 的 LCS
- 若 $x_m \neq y_n$ 且 $z_k \neq y_n$，则 $Z$ 是 $X$ 和 $Y_{n-1}$ 的 LCS

**时间复杂度**：$\Theta(mn)$

**C 语言**：
```c
/* 返回 LCS 长度; X[1..m], Y[1..n] 1-based */
int lcs_length(char X[], int m, char Y[], int n, int c[][n+1]) {
    for (int i = 0; i <= m; i++) c[i][0] = 0;   /* 第 0 列 */
    for (int j = 0; j <= n; j++) c[0][j] = 0;   /* 第 0 行 */
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (X[i] == Y[j])
                c[i][j] = c[i-1][j-1] + 1;
            else
                c[i][j] = (c[i-1][j] > c[i][j-1]) ? c[i-1][j] : c[i][j-1];
    return c[m][n];
}
```

**构造**：从 $b[m,n]$ 回溯，`↖` 表示匹配，打印 $x_i$

## 第 16 章 贪心算法（Greedy Algorithm）

### 核心概念

- 在当前步骤上**总是选取局部最优**方案 → 希望导致全局最优
- 贪心选择是**自顶向下**的：做一次选择后，子问题缩小

### 两个关键要素

| 要素 | 说明 |
| --- | --- |
| **贪心选择性质** | 全局最优解能通过局部最优选择达到 |
| **最优子结构** | 贪心选择 + 剩余子问题最优解 = 原问题最优解 |

### 贪心 vs 动态规划

| | 动态规划 | 贪心算法 |
| --- | --- | --- |
| 选择方式 | 每步依赖子问题解 | 只依赖当前已做选择 |
| 求解方向 | 自底向上 | 自顶向下 |
| 子问题数 | 通常多个子问题 | 通常单个子问题 |

**0-1 背包 vs 零头背包**（PPT 以这两个问题说明贪心与 DP 的区别）：
- **0-1 背包**：物品要么拿要么不拿 → **只能用动态规划**
- **零头背包**：物品可取部分 → **可用贪心算法**（按单位价值排序，依次装）

### 实例一：活动选择问题

**问题**：$n$ 个活动 $[s_i, f_i)$，选最多互不冲突的活动。

**贪心策略**：每次选**最早完成**且与已选活动相容的活动。

**定理 16.1**：对非空子问题 $S_{ij}$，设 $a_m$ 是 $S_{ij}$ 中最早完成的活动，则：
1. $a_m$ 必包含在 $S_{ij}$ 的某个最优解中
2. 子问题 $S_{im}$ 为空集

**递归贪心算法**：
```
RECURSIVE-ACTIVITY-SELECTOR(s, f, i, j)
1  m ← i + 1
2  while m < j and s_m < f_i    ▹ 找第一个与 a_i 相容的活动
3      do m ← m + 1
4  if m < j
5      then return {a_m} ∪ RECURSIVE-ACTIVITY-SELECTOR(s, f, m, j)
6      else return ∅
```

**C 语言**（递归版，s/f 已按完成时间排序）：
```c
/* 从活动 i+1 开始找与活动 i 相容的活动，j 为哨兵 n */
int rec_activity_select(int s[], int f[], int i, int n, int result[], int idx) {
    int m = i + 1;
    while (m < n && s[m] < f[i]) m++;         /* 找第一个相容活动 */
    if (m < n) {
        result[idx] = m;
        return rec_activity_select(s, f, m, n, result, idx + 1);
    }
    return idx;                               /* 返回已选活动数 */
}
```

**迭代贪心算法**：
```
GREEDY-ACTIVITY-SELECTOR(s, f)
1  n ← length[s]
2  A ← {a_1}
3  i ← 1
4  for m ← 2 to n
5      do if s_m ≥ f_i
6          then A ← A ∪ {a_m}
7               i ← m
8  return A
```

**C 语言**（迭代版）：
```c
/* s[0..n-1]和f[0..n-1] 已按完成时间 f 递增排序 */
int greedy_activity_selector(int s[], int f[], int n, int result[]) {
    result[0] = 0;                          /* 第 0 个活动 (最早完成) */
    int i = 0, cnt = 1;
    for (int m = 1; m < n; m++) {
        if (s[m] >= f[i]) {                 /* a_m 与已选活动相容 */
            result[cnt++] = m;
            i = m;
        }
    }
    return cnt;                             /* 返回选中的活动数 */
}
```

**时间复杂度**：$\Theta(n)$（已按完成时间排序）

### 实例二：Huffman 编码

**目标**：用变长二进制编码压缩文件，使总编码长度最小。

**前缀编码**：没有任何编码是另一个编码的前缀。

**文件编码总位数**：$B(T) = \sum_{c \in C} f(c) \cdot d_T(c)$

**Huffman 贪心算法**：
1. 每个字符形成单节点树（频率为权）
2. 合并两个频率最低的树 → 新节点频率为两子节点之和
3. 重复直到剩下一个根节点

**C 语言**（简化版，用数组模拟最小堆选两个最小频度）：
```c
typedef struct { char ch; int freq; int left, right; } HuffNode;

/* 从 nodes[0..size-1] 中找最小频度的两个节点下标 */
void find_two_min(HuffNode nodes[], int size, int *a, int *b) {
    *a = *b = -1;
    int min1 = 2147483647, min2 = 2147483647;
    for (int i = 0; i < size; i++) {
        if (nodes[i].freq < min1) {
            min2 = min1; *b = *a;
            min1 = nodes[i].freq; *a = i;
        } else if (nodes[i].freq < min2) {
            min2 = nodes[i].freq; *b = i;
        }
    }
}

void huffman(int freq[], char chars[], int n) {
    HuffNode nodes[2 * n - 1];              /* n 叶 + n-1 内部 */
    int size = n;
    for (int i = 0; i < n; i++)             /* 初始化叶子 */
        nodes[i] = (HuffNode){chars[i], freq[i], -1, -1};
    for (int i = n; i < 2 * n - 1; i++) {   /* 合并 n-1 次 */
        int a, b;
        find_two_min(nodes, size, &a, &b);
        nodes[i] = (HuffNode){0, nodes[a].freq + nodes[b].freq, a, b};
        nodes[a].freq = nodes[b].freq = 2147483647;  /* 标记已用 */
        size++;
    }
}
```

**算法性质**：
- **贪心选择性质**：最优树中两个最低频字符是最大深度的兄弟叶结点
- **最优子结构**：合并后的问题仍是找最优树

### 16.4 贪心算法的理论基础 — 胚（Matroid）

**定义**：有序对 $M = (S, \mathcal{I})$ 满足：
1. $S$ 是有穷非空集
2. $\mathcal{I}$ 是 $S$ 的独立子集簇，**遗传性**：$B \in \mathcal{I}, A \subseteq B \implies A \in \mathcal{I}$
3. **交换性质**：$A, B \in \mathcal{I}, |A| < |B| \implies \exists x \in B-A, A \cup \{x\} \in \mathcal{I}$

**关键定理**：
- 所有最大独立子集大小相同
- 加权胚的贪心算法返回最优子集

**贪心算法**（加权胚找最大权独立子集）：
```
GREEDY(M, w)
1  A ← ∅
2  将 S 按权值递减排序
3  for each x ∈ S (按序)
4      do if A ∪ {x} ∈ I
5          then A ← A ∪ {x}
6  return A
```

**C 语言**（任务调度版——求最大权的独立任务集）：
```c
/* 任务: id, deadline d[i], 权值 w[i]; 返回早任务集 A 的元素个数 */
int task_schedule(int d[], int w[], int n, int A[]) {
    /* 按权值递减排序 (此处简化为直接操作，实际需先排序) */
    int cnt = 0;
    for (int i = 0; i < n; i++) {             /* 假设已按 w 降序 */
        /* 检查 A ∪ {i} 是否独立: 按 deadline 排序后 N_t(A) ≤ t */
        A[cnt] = i;                           /* 暂时加入 */
        /* 将 A[0..cnt] 按 deadline 排序 */
        for (int a = cnt; a > 0; a--)
            if (d[A[a]] < d[A[a-1]]) {
                int tmp = A[a]; A[a] = A[a-1]; A[a-1] = tmp;
            }
        /* 验证: 对每个位置 t=1..cnt+1, 需 d[A[t-1]] ≥ t */
        int ok = 1;
        for (int t = 1; t <= cnt + 1; t++)
            if (d[A[t-1]] < t) { ok = 0; break; }
        if (ok) cnt++;                        /* 独立，保留 */
    }
    return cnt;                               /* 返回早任务数 */
}
```

### 16.5 任务调度问题

**问题**：单处理器上单位时间任务调度，每个任务有截止期 $d_i$ 和罚款 $w_i$，最小化总罚款。

**转化**：最小化迟任务罚款 ⇔ 最大化早任务权值和（找权最大的独立任务集）

**独立任务集判定**（引理 16.12）：$A$ 独立 $\iff$ $\forall t, N_t(A) \le t$（$N_t(A)$ 是截止期 $\le t$ 的任务数）

**结论**：$(S, \mathcal{I})$ 是胚 → 贪心算法可求最优调度。

## 第 24 章 单源最短路径算法

### 基本概念

- **带权有向图** $G=(V,E)$，权值 $w: E \to \mathbb{R}$
- 路径 $p$ 的权：$w(p) = \sum w(v_i, v_{i+1})$
- **最短路径最优子结构**：最短路径的子路径也是最短路径
- 最短路径不含回路（至多 $|V|-1$ 条边）
- **前趋图** $G_\pi$ 是一棵以 $s$ 为根的最短路径树

### 松弛技术（Relaxation）

```
RELAX(u, v, w)
1  if d[v] > d[u] + w(u, v)
2      then d[v] ← d[u] + w(u, v)
3           π[v] ← u
```

**图的 C 语言表示**：用邻接矩阵 `int G[V][V]`，不存在的边用 `INF`（大数）表示。

### 算法对比

| 算法 | 适用条件 | 时间复杂度 |
| --- | --- | --- |
| **Bellman-Ford** | 一般图（允许负权边，检测负权回路） | $O(VE)$ |
| **DAG 最短路径** | 有向无回路图 | $O(V+E)$ |
| **Dijkstra** | 非负权图 | 取决于优先队列实现 |

### Bellman-Ford 算法

**思想**：对每条边进行 $|V|-1$ 轮松弛，第 $|V|$ 轮检测负权回路。

```
BELLMAN-FORD(G, w, s)
1  INITIALIZE-SINGLE-SOURCE(G, s)
2  for i ← 1 to |V[G]| - 1
3      do for each edge (u, v) ∈ E[G]
4          do RELAX(u, v, w)
5  for each edge (u, v) ∈ E[G]
6      do if d[v] > d[u] + w(u, v)
7          then return FALSE    ▹ 存在负权回路
8  return TRUE
```

**C 语言**（邻接矩阵版）：
```c
#define INF 2147483647
#define V 100   /* 顶点数上限 */

/* 用邻接矩阵 G[V][V]; d[0..V-1] 存最短路估计; 返回 1=成功, 0=有负权回路 */
int bellman_ford(int G[][V], int n, int s, int d[], int pi[]) {
    /* 初始化 */
    for (int i = 0; i < n; i++) { d[i] = INF; pi[i] = -1; }
    d[s] = 0;
    /* V-1 轮松弛所有边 */
    for (int k = 1; k < n; k++)
        for (int u = 0; u < n; u++)
            for (int v = 0; v < n; v++)
                if (G[u][v] != INF && d[u] != INF && d[v] > d[u] + G[u][v]) {
                    d[v] = d[u] + G[u][v];
                    pi[v] = u;
                }
    /* 检测负权回路 */
    for (int u = 0; u < n; u++)
        for (int v = 0; v < n; v++)
            if (G[u][v] != INF && d[u] != INF && d[v] > d[u] + G[u][v])
                return 0;
    return 1;
}
```

### DAG 最短路径

**思想**：先拓扑排序，再按拓扑序依次松弛每条边。

**伪码**：
```
DAG-SHORTEST-PATHS(G, w, s)
1  拓扑排序 G 的顶点
2  INITIALIZE-SINGLE-SOURCE(G, s)
3  for each vertex u (按拓扑序)
4      do for each vertex v ∈ Adj[u]
5          do RELAX(u, v, w)
```

**C 语言**（邻接矩阵，假设顶点已按拓扑序编号 0..n-1）：
```c
/* topo[0..n-1] 为拓扑排序结果; G[V][V] 邻接矩阵 */
void dag_shortest_paths(int G[][V], int n, int topo[], int s,
                        int d[], int pi[]) {
    for (int i = 0; i < n; i++) { d[i] = INF; pi[i] = -1; }
    d[s] = 0;
    for (int i = 0; i < n; i++) {
        int u = topo[i];                     /* 按拓扑序取顶点 */
        for (int v = 0; v < n; v++)
            if (G[u][v] != INF && d[u] != INF && d[v] > d[u] + G[u][v]) {
                d[v] = d[u] + G[u][v];
                pi[v] = u;
            }
    }
}
```

**时间复杂度**：$\Theta(V+E)$

### Dijkstra 算法

**前提**：所有边权非负。

**思想**：维护集合 $S$（已确定最短路径的顶点），每次选 $d$ 最小的顶点加入 $S$，松弛其出边。

```
DIJKSTRA(G, w, s)
1  INITIALIZE-SINGLE-SOURCE(G, s)
2  S ← ∅
3  Q ← V[G]                    ▹ 最小优先队列
4  while Q ≠ ∅
5      do u ← EXTRACT-MIN(Q)
6          S ← S ∪ {u}
7          for each vertex v ∈ Adj[u]
8              do RELAX(u, v, w)
```

**C 语言**（数组实现，$O(V^2)$）：
```c
void dijkstra(int G[][V], int n, int s, int d[], int pi[]) {
    int visited[V] = {0};                     /* visited[v]=1 表示 v 在 S 中 */
    for (int i = 0; i < n; i++) { d[i] = INF; pi[i] = -1; }
    d[s] = 0;
    for (int cnt = 0; cnt < n; cnt++) {
        /* EXTRACT-MIN: 选最小的未确定顶点 */
        int u = -1, min = INF;
        for (int i = 0; i < n; i++)
            if (!visited[i] && d[i] < min) { min = d[i]; u = i; }
        if (u == -1) break;                   /* 剩余节点不可达 */
        visited[u] = 1;                       /* 加入 S */
        /* RELAX 所有出边 */
        for (int v = 0; v < n; v++)
            if (G[u][v] != INF && !visited[v] && d[v] > d[u] + G[u][v]) {
                d[v] = d[u] + G[u][v];
                pi[v] = u;
            }
    }
}
```

**时间复杂度取决于优先队列实现**：

| 优先队列实现 | 时间复杂度 |
| --- | --- |
| 数组 | $O(V^2 + E)$ |
| 二叉最小堆 | $O((V+E) \log V)$ |
| 斐波那契堆 | $O(V \log V + E)$ |

- $f(n)$ 多项式大于 + 规则性条件 → $\Theta(f(n))$
