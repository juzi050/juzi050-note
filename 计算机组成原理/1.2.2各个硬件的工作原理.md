# 主存储器

![图片展示了主存储器的结构。主存储器由存储体、MAR（Memory Address Register）和MDR（Memory Data Register）组成。存储体是核心部分，用于按地址存储数据。MAR位数反映存储单元个数，MDR位数等于存储字长。该图与上文介绍的主存储器相关，直观呈现了主存储器各组成部分及其位置关系，帮助理解主存储器的工作原理。](../assets/images/计算机组成原理-1.2.2各个硬件的工作原理-01-46ce30a90350.png)

## 存储体**：**

数据在存储体内按地址存储

![图片展示了主存储器中存储体和存储单元的关系。左侧是主存储器，包含存储体、MAR（Memory Address Register）和MDR（Memory Data Register）。右侧是存储体，以表格形式呈现，每个单元对应一个地址，如0、1、2等，每个地址对应一个存储单元。图中用红色箭头和文字标注，强调数据在存储体内按地址存储，每个地址对应一个存储单元，直观呈现了主存储器中地址与存储单元的对应关系。](../assets/images/计算机组成原理-1.2.2各个硬件的工作原理-02-6898fba8a82b.png)

**存储单元**：每个存储单元存放一串二进制代码

**存储字（word）**：存储单元中二进制代码的组合

**存储字长**：存储单元中二进制代码的位数

**存储元**：存储二进制的电子元件，每个存储元可存1bit

## MAR（Memory Address Register）

MAR的位数反映存储单元的个数

## MDR（Memory Data Register）

MDR位数=存储字长

# 运算器

用于实现算术运算、逻辑运算

![图片展示了运算器的结构。运算器由ACC、MQ、X、ALU四部分组成，ACC、MQ、X分别位于运算器的上、右、下侧，ALU位于中间。ACC、MQ、X与ALU之间有箭头连接，ACC与MQ之间也有箭头连接，表明它们之间有数据传输关系。该图与文档中运算器部分内容对应，直观呈现了运算器各部分的连接关系及数据流动情况。](../assets/images/计算机组成原理-1.2.2各个硬件的工作原理-03-6536adcd56c2.png)

- ACC：累加器，用于存放操作数，或运算结果
- MQ：累商寄存器，在乘、除运算时，用于存放操作数或运算结果
- X：通用的操作数寄存器，用于存放操作数
- ALU：算术逻辑单元，通过内部复杂电路实现算术运算、逻辑运算

|  | 加 | 减 | 乘 | 除 |
|-|-|-|-|-|
| ACC | 被加数、和 | 被减数、差 | 乘积高位 | 被除数、余数 |
| MQ |  |  | 乘数、乘积低位 | 商 |
| X | 加数 | 减数 | 被乘数 | 除数 |

# 控制器

![图片展示了计算机控制器的结构。控制器被红圈突出显示，内部包含CU（控制单元）、IR（指令寄存器）和PC（程序计数器）。CU位于下方，IR和PC分别位于其两侧。该图与上下文紧密相关，上下文介绍了控制器的工作原理，CU分析指令给出控制信号，IR存放当前指令，PC存放下一条指令地址并有自动加1功能，此图直观呈现了控制器内部各部分的布局。](../assets/images/计算机组成原理-1.2.2各个硬件的工作原理-04-e59d2bded59c.png)

- CU：控制单元，分析指令，给出控制信号
- IR：指令寄存器，存放当前执行的指令
- PC：程序计数器，存放下一条指令地址，有自动加1功能

![图片展示了计算机完成一条指令的三个步骤。左侧用红色曲线标注“完成一条指令”，右侧列出三个步骤及其对应硬件：取指令对应PC（程序计数器），分析指令对应IR（指令寄存器），执行指令对应CU（控制单元）。该图与文档中控制器部分内容相关，直观呈现了控制器在指令执行过程中的作用，帮助理解控制器分析指令、给出控制信号以完成指令执行的流程。](../assets/images/计算机组成原理-1.2.2各个硬件的工作原理-05-a4920444bc1f.png)
