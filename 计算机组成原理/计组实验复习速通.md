# 2 指令编码

|   知识点   |             超简单解释             |                      必须背住                       |                     考试怎么做                     |
| :-----: | :---------------------------: | :---------------------------------------------: | :-------------------------------------------: |
| 什么是指令编码 | 汇编指令是人能看懂的写法，机器码是CPU真正执行的二进制。 |             一条汇编指令最终会变成一串32位二进制机器码。             |         题目给机器码，就要反推出汇编；题目给汇编，就要写出机器码。         |
|  指令长度   |     RISC-V 基本整数指令通常是32位。      |              32位 = 4字节 = 8个十六进制数。               |      例如 FFD00293H 是8位十六进制，所以正好是一条32位指令。       |
|   字段    |   一条32位指令不是随便写的，而是被分成几个小格子。   |   常见字段有 opcode、rd、funct3、rs1、rs2、imm、funct7。    |            机器码分析题就是把这32位按格式切成这些字段。            |
| opcode  |   opcode 是操作码，决定这条指令属于哪一大类。   |                 最低7位永远是 opcode。                 |          反推机器码时，第一步永远先看最低7位 opcode。           |
|   rd    |     rd 是目的寄存器，也就是结果写到哪里。      |     rd 占5位，因为寄存器 x0~x31 一共32个，5位刚好能表示32个编号。     |            看到 rd=00101，就说明结果写入 x5。            |
|   rs1   |         rs1 是第一个源寄存器。         |                    rs1 也是5位。                    |            例如 rs1=00000 表示从 x0 取数。            |
|   rs2   |         rs2 是第二个源寄存器。         |                  R型和S型常用 rs2。                   |           sll x7,x6,x5 中，x5 就是 rs2。           |
| funct3  |  funct3 是3位功能码，用来进一步区分具体指令。   | 同一个 opcode 下，要靠 funct3 判断是 addi、andi、sw 还是 sll。 | 例如 I型运算中 funct3=000 是 addi，funct3=111 是 andi。 |
| funct7  |    funct7 是7位功能码，主要用于R型指令。    |       add 和 sub 的 funct3 一样，靠 funct7 区分。        |  add 的 funct7=0000000，sub 的 funct7=0100000。   |
|   imm   |   imm 是立即数，也就是指令里直接写出来的常数。    |                  立即数经常需要符号扩展。                   |            如果立即数最高位是1，通常要按负数补码理解。             |

## RISC-V 指令格式总表

- **指令字长**：固定为 32 位，必须按 4 字节对齐。
- **指令格式**：常见 6 种：R 型、I 型、S 型、B 型、U 型、J 型。

| 指令格式类型 | inst[31:25]<br>7 位 | inst[24:20]<br>5 位 | inst[19:15]<br>5 位 | inst[14:12]<br>3 位 | inst[11:7]<br>5 位 | inst[6:0]<br>7 位 |    指令类型    |
| :----: | :----------------: | :----------------: | :----------------: | :----------------: | :---------------: | :--------------: | :--------: |
|  R 型   |       funct7       |        rs2         |        rs1         |       funct3       |        rd         |      opcode      |    运算指令    |
|  I 型   |     imm[11:5]      |      imm[4:0]      |        rs1         |       funct3       |        rd         |      opcode      | 装数 / 立即数运算 |
|  S 型   |     imm[11:5]      |        rs2         |        rs1         |       funct3       |     imm[4:0]      |      opcode      |    存数指令    |
|  B 型   |    imm[12,10:5]    |        rs2         |        rs1         |       funct3       |    imm[4:1,11]    |      opcode      |   条件分支指令   |
|  U 型   |     imm[31:25]     |     imm[24:20]     |     imm[19:15]     |     imm[14:12]     |        rd         |      opcode      |    大立即数    |
|  J 型   |    imm[20,10:5]    |    imm[4:1,11]     |     imm[19:15]     |     imm[14:12]     |        rd         |      opcode      |  无条件跳转指令   |

- **opcode：**7 位，固定在指令字最低 7 位 inst[6:0]，指定指令的大类 / 格式。
- **寄存器字段：**rs1、rs2、rd 都是 5 位，位置基本固定；5 位刚好表示 x0～x31。
- **立即数字段：**主要有 12 位和 20 位两种，立即数最高位通常来自 inst[31]，用于符号扩展。

| 格式 | 二进制排列 | 用来干什么 | 常见指令 |
| :---: | :---: | :---: | :---: |
| R型 | funct7 + rs2 + rs1 + funct3 + rd + opcode | 寄存器和寄存器运算。 | add、sub、sll、and、or |
| I型 | imm[11:0] + rs1 + funct3 + rd + opcode | 寄存器和立即数运算，或者从内存读数据。 | addi、andi、ori、lw、jalr |
| S型 | imm[11:5] + rs2 + rs1 + funct3 + imm[4:0] + opcode | 把寄存器数据写入内存。 | sw、sh、sb |
| U型 | imm[31:12] + rd + opcode | 装入高20位立即数。 | lui、auipc |

| 常见指令 | 格式 | opcode | funct3 | funct7 | 含义 |
| :---: | :---: | :---: | :---: | :---: | :---: |
| addi | I型 | 0010011 | 000 | 无 | rd = rs1 + imm |
| andi | I型 | 0010011 | 111 | 无 | rd = rs1 & imm |
| lw | I型 | 0000011 | 010 | 无 | rd = Mem[rs1 + imm] |
| sw | S型 | 0100011 | 010 | 无 | Mem[rs1 + imm] = rs2 |
| sll | R型 | 0110011 | 001 | 0000000 | rd = rs1 << rs2 |
| lui | U型 | 0110111 | 无 | 无 | rd = imm20 << 12 |

| 机器码反推汇编步骤 | 具体操作 | 例子 |
| :---: | :---: | :---: |
| 第1步 | 把十六进制机器码转成32位二进制。 | FFD00293H → 11111111110100000000001010010011 |
| 第2步 | 看最低7位 opcode。 | 0010011，说明是I型立即数运算。 |
| 第3步 | 按I型格式切字段。 | imm[11:0] + rs1 + funct3 + rd + opcode |
| 第4步 | 看 funct3 判断具体指令。 | funct3=000，所以是 addi。 |
| 第5步 | 把寄存器字段转成 x编号。 | rd=00101 是 x5，rs1=00000 是 x0。 |
| 第6步 | 立即数按补码转十进制。 | imm=111111111101，12位补码是 -3。 |
| 结论 | 写出汇编助记符。 | FFD00293H = addi x5,x0,-3。 |

# 3 汇编程序分析

| 要看什么 | 超简单解释 | 分析方法 | 例子 |
| :---: | :---: | :---: | :---: |
| 看指令名 | 指令名决定这条语句要干什么。 | 先判断是运算、移位、访存，还是跳转。 | addi 是加立即数，sw 是存内存，sll 是左移。 |
| 看 rd | rd 是结果写到哪里。 | 有 rd 的指令一般会改变寄存器。 | addi x5,x0,3 改变 x5。 |
| 看 rs1 | rs1 是第一个参与计算的寄存器。 | 计算前先查 rs1 的当前值。 | addi x5,x6,3 中，要先知道 x6 是多少。 |
| 看 rs2 | rs2 是第二个参与计算的寄存器，或者要存入内存的数据来源。 | R型运算和 sw 常用 rs2。 | sw x7,0(x6) 中，x7 是要写入内存的数据。 |
| 看 imm | imm 是常数，也可能是地址偏移量。 | 访存地址通常是 rs1 + imm。 | sw x7,0(x6) 的地址是 x6+0。 |
| 看标签 label | 标签是跳转目标。 | 如果分支跳回前面的标签，通常是循环。 | bne x5,x6,loop 表示不相等就跳回 loop。 |

| 指令 | 你应该翻译成什么 | 例子 | 执行结果 |
| :---: | :---: | :---: | :---: |
| addi | 加一个立即数。 | addi x5,x0,3 | x5 = 0 + 3 = 3 |
| andi | 按位与一个立即数。 | andi x7,x0,-0x100 | x7 = 0 & (-0x100) = 0 |
| lui | 把立即数放到高20位，低12位补0。 | lui x8,0x100 | x8 = 0x100 << 12 = 0x100000 |
| sll | 左移。 | sll x7,x6,x5 | x7 = x6 左移 x5 位 |
| lw | 从内存读数据到寄存器。 | lw x5,0(x6) | x5 = Mem[x6+0] |
| sw | 把寄存器数据写入内存。 | sw x7,0(x6) | Mem[x6+0] = x7 |
| beq | 两个寄存器相等就跳转。 | beq x5,x6,L1 | 如果 x5==x6，跳到 L1。 |
| bne | 两个寄存器不相等就跳转。 | bne x5,x6,loop | 如果 x5!=x6，跳到 loop。 |

| 分析汇编程序的固定套路 | 具体做法 |
| :---: | :---: |
| 第1步：列初值 | 把题目给出的寄存器初值、内存初值写下来。没有给出的不要乱猜。 |
| 第2步：逐条执行 | 一条指令一条指令往下算，每执行一条就更新寄存器或内存。 |
| 第3步：遇到分支判断条件 | 看到 beq、bne、blt、bge，要判断条件成立不成立。 |
| 第4步：识别循环 | 如果跳转目标在前面，说明可能是循环，要看循环变量怎么变化。 |
| 第5步：总结功能 | 不要只写“执行了加法和存储”，要写成“把某个值计算出来后存到某地址”。 |

# 4 C转汇编

| C语言写法 | 你脑子里要怎么想 | RISC-V写法 | 例子 |
| :---: | :---: | :---: | :---: |
| a = 3; | 把常数3放进保存a的寄存器。 | addi 目标寄存器,x0,3 | addi x5,x0,3 |
| a = b; | 把b复制给a。 | addi 目标寄存器,源寄存器,0 | addi x5,x6,0 |
| a = b + c; | 两个变量相加。 | add 保存a的寄存器,保存b的寄存器,保存c的寄存器 | add x5,x6,x7 |
| a = b + 3; | 变量加常数。 | addi 保存a的寄存器,保存b的寄存器,3 | addi x5,x6,3 |
| a = b - c; | 两个变量相减。 | sub 保存a的寄存器,保存b的寄存器,保存c的寄存器 | sub x5,x6,x7 |
| a = b << 3; | 左移3位，相当于乘以8。 | slli 或 sll | slli x5,x6,3 |
| A[0] = a; | 把a写入数组A的第0个元素。 | sw 保存a的寄存器,0(数组基地址寄存器) | sw x5,0(x6) |
| a = A[0]; | 从数组A第0个元素读到a。 | lw 保存a的寄存器,0(数组基地址寄存器) | lw x5,0(x6) |

| 数组访问 | 超简单解释 | 例子 |
| :---: | :---: | :---: |
| 数组基地址 | 数组不是一个数，而是一片连续内存。数组名可以理解为起始地址。 | A 的基地址假设放在 x6。 |
| int类型大小 | 一个 int 通常占4字节。 | A[0] 偏移0，A[1] 偏移4，A[2] 偏移8。 |
| A[i]地址 | A[i] 的地址 = A的基地址 + i*4。 | 如果 i=3，则偏移是12。 |
| 读数组 | 先算地址，再 lw。 | lw x5,12(x6) 表示 x5=A[3]。 |
| 写数组 | 先算地址，再 sw。 | sw x5,12(x6) 表示 A[3]=x5。 |

| C转汇编固定套路 | 操作 |
| :---: | :---: |
| 第1步 | 先给每个变量安排一个寄存器。例如 a 用 x5，b 用 x6，c 用 x7。 |
| 第2步 | 看右边表达式，先算右边。 |
| 第3步 | 如果是常数，用 addi；如果是两个变量，用 add/sub/sll 等R型指令。 |
| 第4步 | 如果涉及数组或指针，先算地址，再用 lw 或 sw。 |
| 第5步 | 最后把结果放到左边变量对应的寄存器或内存位置。 |

# 5 CPU设计

| 阶段 | 这一阶段干什么 | 你可以这样写 | 哪些指令需要 |
| :---: | :---: | :---: | :---: |
| 取指 IF | 根据 PC 到指令存储器取出当前指令。 | IR ← InstMem[PC]，PC ← PC + 4 | 所有指令都需要取指。 |
| 译码 ID | 拆指令字段，读寄存器，生成立即数，判断指令类型。 | 读 rs1、rs2；ImmGen 产生立即数；控制器译码。 | 所有指令都需要译码。 |
| 执行 EX | 让 ALU 工作，完成运算或地址计算。 | addi 做加法，sll 做左移，sw 做地址计算。 | addi、sll、sw 都需要执行阶段。 |
| 访存 MEM | 访问数据存储器。 | sw 要写内存，lw 要读内存。 | sw 必须访存；addi、sll 不需要数据访存。 |
| 写回 WB | 把结果写回寄存器堆。 | Reg[rd] ← ALU结果 或 Mem读出数据。 | addi、sll 需要写回；sw 不写回。 |

| 指令 | 执行流程 | 每步解释 |
| :---: | :---: | :---: |
| addi rd,rs1,imm | 取指 → 译码 → 执行 → 写回 | 先取出指令；读 rs1 和 imm；ALU 算 rs1+imm；结果写入 rd。 |
| sll rd,rs1,rs2 | 取指 → 译码 → 执行 → 写回 | 先取出指令；读 rs1 和 rs2；ALU 做左移；结果写入 rd。 |
| sw rs2,imm(rs1) | 取指 → 译码 → 执行 → 访存 | 先取出指令；读 rs1、rs2 和 imm；ALU 算地址 rs1+imm；把 rs2 写入该地址。 |

| 状态 | 状态名 | 要做的事 | 下一状态 |
| :---: | :---: | :---: | :---: |
| S0 | 取指 | IR ← InstMem[PC]，PC ← PC+4 | S1 译码 |
| S1 | 译码 | 读寄存器 rs1、rs2，生成立即数，判断指令类型。 | 如果是 addi/sll/sw，都进入执行状态。 |
| S2 | 执行 addi | ALUOut ← Reg[rs1] + imm | S3 写回 addi |
| S3 | 写回 addi | Reg[rd] ← ALUOut | S0 取下一条指令 |
| S4 | 执行 sll | ALUOut ← Reg[rs1] << Reg[rs2] | S5 写回 sll |
| S5 | 写回 sll | Reg[rd] ← ALUOut | S0 取下一条指令 |
| S6 | 执行 sw | ALUOut ← Reg[rs1] + imm | S7 存储 |
| S7 | 存储 sw | DataMem[ALUOut] ← Reg[rs2] | S0 取下一条指令 |

| 控制信号 | 超简单解释 | 什么时候有效 |
| :---: | :---: | :---: |
| PCWrite | 允许 PC 更新。 | 取指阶段有效，因为要 PC ← PC+4。 |
| IRWrite | 允许 IR 装入新指令。 | 取指阶段有效。 |
| RegWrite | 允许写寄存器堆。 | addi、sll 写回阶段有效；sw 无效。 |
| MemWrite | 允许写数据存储器。 | sw 的存储阶段有效。 |
| ALUSrcA | 选择 ALU 的第一个输入。 | 取指时可选 PC；执行时通常选 rs1。 |
| ALUSrcB | 选择 ALU 的第二个输入。 | addi/sw 选 imm；sll 选 rs2；PC+4 时选常数4。 |
| ALUOp | 控制 ALU 做什么运算。 | addi/sw 用加法；sll 用左移。 |
| ResultSrc | 选择写回寄存器的数据来源。 | addi/sll 选择 ALU 结果；lw 选择内存数据。 |

# 6 Verilog与测试

| 模块 | 它是什么 | 你要会什么 | 常见写法 |
| :---: | :---: | :---: | :---: |
| PC | 程序计数器，保存当前指令地址。 | 会写复位清零、时钟上升沿更新。 | if(reset) pc<=0; else if(PCWrite) pc<=next_pc; |
| IR | 指令寄存器，保存当前正在执行的指令。 | 只有 IRWrite 有效时才更新。 | if(IRWrite) ir<=inst; |
| 寄存器堆 | 32个通用寄存器 x0~x31。 | 两个读口，一个写口；x0 永远为0。 | if(RegWrite && rd!=0) regs[rd]<=wd; |
| ALU | 算术逻辑单元。 | 会根据控制信号做加法、减法、与、或、左移。 | case(ALUOp) ADD: y=a+b; SLL: y=a<<b[4:0]; endcase |
| 立即数扩展器 | 把指令里的 imm 变成32位。 | 会区分 I型、S型、U型。 | I型取 instr[31:20]；S型拼 instr[31:25] 和 instr[11:7]。 |
| 数据存储器 | 保存程序运行中的数据。 | sw 时写入，lw 时读出。 | if(MemWrite) mem[addr]<=write_data; |
| MUX | 多路选择器。 | 根据控制信号选择一路数据。 | assign y = sel ? b : a; |
| 控制器 | CPU的大脑，负责产生控制信号。 | 根据当前状态和 opcode/funct 判断下一状态。 | case(state) S0: ... S1: ... endcase |

| 测试程序 | 执行结果 | 为什么 |
| :---: | :---: | :---: |
| addi x5,x0,3 | x5 = 3 | x0 永远是0，所以 x5=0+3。 |
| addi x6,x0,0x100 | x6 = 0x100 | x6=0+0x100。 |
| sll x7,x6,x5 | x7 = 0x800 | x6=0x100，x5=3，所以 x7=0x100<<3=0x800。 |
| sw x7,0(x6) | Mem[0x100] = 0x800 | 存储地址是 x6+0=0x100，写入数据是 x7=0x800。 |

| 如何判断测试通过 | 判断依据 |
| :---: | :---: |
| 看寄存器 | x5 应该等于 3，x6 应该等于 0x100，x7 应该等于 0x800。 |
| 看内存 | 地址 0x100 处的数据应该是 0x800。 |
| 看PC | 每执行一条32位指令，PC 通常加4。 |
| 看控制信号 | addi/sll 写回时 RegWrite 应该有效；sw 写内存时 MemWrite 应该有效。 |
| 看波形 | 波形中每条指令应按取指、译码、执行、写回或访存顺序变化。 |

# 7 拓展题

| 题型 | 超简单解释 | 答题步骤 | 例子 |
| :---: | :---: | :---: | :---: |
| 控制信号故障 | 某根控制线坏了，永远是0或永远是1。 | 先看它控制哪个部件，再看哪些指令必须用这个部件。 | 如果 MemWrite 恒0，则 sw 不能写内存。 |
| MUX选择错误 | MUX 本来要在两个数据中选一个，现在固定只能选某一路。 | 判断正确执行时需要哪一路，如果需要另一条路就错。 | 如果 ALU 第二输入不能选 imm，则 addi 和 sw 地址计算会出错。 |
| 寄存器写使能故障 | RegWrite 如果坏了，寄存器可能写不进去。 | 所有需要写回 rd 的指令都受影响。 | addi、sll 受影响；sw 本来不写寄存器，所以不受这个影响。 |
| 存储器写使能故障 | MemWrite 控制数据存储器能不能写。 | 只要是存储类指令就要看它。 | sw 受影响；addi、sll 不需要写内存。 |
| 删除冗余部件 | 如果考试只要求支持几条指令，有些部件用不到，可以删。 | 先列指令需要什么，再删掉完全不用的。 | 只支持 addi、sll、sw，就不需要分支判断器和跳转目标选择。 |
| ALU替代PC+4加法器 | 原来有专门加法器算 PC+4，现在改成让 ALU 算。 | 取指阶段 ALU 输入选 PC 和常数4，ALU结果写回 PC。 | 好处是少一个加法器；坏处是取指阶段 ALU 被占用。 |
| ALU替代分支加法器 | 原来分支目标地址由专门加法器计算，现在也让 ALU 算。 | 分支时 ALU 计算 PC+imm，然后结果作为新PC。 | 可能会增加状态，因为 ALU 同一时间只能做一个运算。 |

| 拓展题万能答题模板 | 写法 |
| :---: | :---: |
| 第1句 | 说明这个信号或部件的作用。 |
| 第2句 | 说明哪些指令会用到它。 |
| 第3句 | 说明坏了或删了以后，这些指令会出现什么错误。 |
| 第4句 | 说明哪些指令不依赖它，所以仍然可以正常执行。 |
| 第5句 | 如果是结构改动题，再补充数据通路怎么改、控制信号怎么改、执行流程是否增加状态。 |
