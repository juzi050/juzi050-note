# 5.2 指令执行过程
## 指令周期
CPU从主存中每取出并执行一条指令所需的全部时间
**指令周期**通常用若干**机器周期**表示，机器周期又叫**CPU周期**。
一个**机器周期**包含若干**时钟周期**（也称作节拍，T周期或者CPU时钟周期），是CPU操作的**最基本单元**
![[Pasted image 20260618204311.png]]
![[Pasted image 20260618204551.png]]
## 指令周期流程
![[Pasted image 20260618205045.png]]
四个工作周期都有CPU访存操作，只是访存的目的不同。
1. **取指周期**：取指令
2. **间址周期**：取有效地址
3. **执行周期**：取操作数
4. **中断周期**：保存程序断点
## 指令周期的数据流
### 取指周期
1. 当前指令地址送至MAR，记作：$(PC)\to MAR$
2. CU发出控制信号，经过控制总线传到主存，这里是读信号，记作：$1\to R$
3. 把MAR所指主存中的内容经过数据总线送入MDR，记作：$M(MAR)\to MDR$
4. 将MDR中的内容（此时是指令）送入IR，记作：$(MDR)\to IR$
5. CU发出控制信号，形成下一条指令地址，记作：$(PC)+1\to PC$
![[Pasted image 20260618205521.png]]
### 间址周期
1. 将指令的地址码送入MAR，记作：$Ad(IR)\to MAR$或者$Ad(MDR)\to MAR$
2. CU发出控制信号，启动主存做读操作，记作：$1\to R$
3. 将MAR所指主存中的内容经数据总线送入MDR，记作：$M(MAR)\to MDR$
4. 将有效地址送至指令的地址码字段，记作：$(MDR)\to Ad(IR)$

![[Pasted image 20260618210237.png]]
### 执行周期
没有统一的数据流向
### 中断周期
暂停当前任务去完成其他任务。为了能够恢复当前任务，需要保存断点。一般用堆栈来存断点。SP表示栈顶地址，假设SP指向栈顶元素，进栈操作是先修改指针，后存入数据。
类比函数调用。
1. CU控制将SP减1，修改后的地址送入MAR，记作：$(SP)-1\to SP，(SP)\to MAR$
   本质上是将断点存入某个存储单元，假设其地址为a，故可记作：$a\to MAR$
2. CU发出控制信号，启动主存做写操作，记作：$1\to W$
3. 将断点（PC内容）送入MDR，记作：$(PC)\to MDR$
4. CU控制将中断服务程序的入口地址（由向量地址形成部件产生）送入PC，记作：$向量地址\to PC$
![[Pasted image 20260618211832.png]]
## 指令执行方案
### 单指令周期
对所有指令选用相同的执行时间来完成。
指令之间**串行执行**。
指令周期取决于执行时间最长的指令的执行时间。
**优点**：可以使用固定的时钟周期，硬件设计简单
**缺点**：指令周期取决于执行时间最长的指令的执行时间
### 多指令周期
对不同类型的指令选用不同的执行步骤来完成。
指令之间**串行执行**。
可选用不同个数的时钟周期来完成不同指令的执行过程
**优点**：提升效率
**缺点**：需要更复杂的硬件设计
### 流水线方案
在每一个时钟周期启动一条周期，尽量让多条指令同时运行，但各自处在不同的执行步骤中。
指令之间**并行执行**。
# 5.3.1 数据通道-单总线结构
**数据通道**：数据在功能部件之间传送的路径
![[Pasted image 20260619134830.png]]
## 寄存器之间数据传送
**举例**：
把PC内容送至MAR，实现传送操作的流程及控制信号为：
1. $(PC)\to Bus$ ：PCout有效，PC内容送总线
2. $Bus\to MAR$：MARin有效，总线内容送MAR
又可以写作：$(PC)\to Bus\to MAR$ 或者 $PC\to Bus\to MAR$
## 主存与CPU之间的数据传送
**举例**：
CPU从主存中读取指令，实现传送操作的流程及控制信号为：
1. $(PC)\to Bus\to MAR$ ：$PC_{out}和MAR_{in}$有效，现行指令地址$\to MAR$
2. $1\to R$：CU发读命令（通过控制总线发出）
3. $MEM(MAR)\to MDR$：$MDR_{inE}$有效
4. $MDR\to Bus\to IR$：$MDR_{out}和IR_{in}有效$，现行指令$\to IR$
## 执行算术或逻辑运算
**举例**：
一条加法指令，微操作序列及控制信号为：
1. $Ad(IR)\to Bus\to MAR$：$MDR_{out}和MAR_{in}$有效
2. $1\to R$：CU发出读命令
3. $MEM(MAR)\to数据线\to MDR$：$MDR_{inE}$有效
4. $MDR\to Bus\to Y$：$MDR_{out}和Y_{in}有效，操作数\to Y$
5. $(ACC)+(Y)\to Z$：$ACC_{out}和ALU_{in}有效，CU向ALU发送加命令$
6. $Z\to ACC$：$Z_{out}和ACC_{in}有效，结果\to ACC$

# 5.4.3 微指令的设计
**名词解释**

| 名词     | 含义          |
| ------ | ----------- |
| 相容性微命令 | 可以并行完成的微命令  |
| 互斥性微命令 | 不允许并行完成的微命令 |

## 微指令的格式
### 水平型微指令
一条指令能定义**多个**可并行的微命令
基本格式：操作控制+顺序控制
![[Pasted image 20260620114504.png]]
**优点**：微程序短，执行速度快
**缺点**：微指令长，编写微程序较麻烦
### 垂直型微指令
一条微指令只能定义**一个**微命令，由微操作码字段规定具体功能
基本格式：微操作码（μOP）+目的地址（Rd）+源地址（Rs）
![[Pasted image 20260620114512.png]]
**优点**：微指令短、简单、规整、便于编写微程序
**缺点**：微程序长，执行速度慢，工作效率低
### 混合型微指令
在垂直型的基础上增加一些不太复杂的并行操作
微指令较短，仍便于编写，微程序也不长，执行速度加快
## 微指令的编码方式（控制方式）
### 直接编码（直接控制）方式
每一位代表一个微操作命令
![[Pasted image 20260620114833.png]]
### 字段直接编码方式
将微指令的控制字段分成若干段，每段经过译码后发出控制信号
**分段原则**：
1. 互斥性微命令分在同一段内，相容性微命令分在不同段内
2. 每个小段包含的信息位不能太多
3. 每个小段还要留出一个状态，用于表示本字段不发出任何微命令
![[Pasted image 20260620114933.png]]
### 字段间接编码
一个字段中的某些微命令需要另一个字段中的某些微命令来解释，而不是靠该字段直接译码发出的微命令
![[Pasted image 20260620115459.png]]

2. 根据机器指令的**操作码**形成
	1. 当机器指令取至指令寄存器后，微指令的地址由操作码经微地址形成部件形成
3. **增量计数器法**
	1. $(CMAR)+1\to CMAAR$
4. 分支转移
	1. 转移方式：指明判断条件。转移地址：指明转移成功后的去向
	2. ![[Pasted image 20260620120003.png]]
5. 通过测试网络
6. 由硬件产生微程序入口地址
	1. **第一条指令**：由专门的硬件产生
	2. **中断周期**：由硬件产生中断周期微程序首地址
# 5.4.4 微程序控制单元的设计
## 设计步骤
1. 分析每个阶段的微操作序列
2. 写出对应机器指令的微操作命令及节拍安排
	1. 写出每个周期所需要的微操作（参照硬布线）
	2. 补充微程序控制器特有的微操作
		1. 取指周期：
		   $Ad(CMDR)\to CMAR$：每条微指令结束后都要执行
		   $OP(IR)\to微地址形成部件\to CMAR$：取指周期的最后一条微指令完成后，要根据指令操作码确定其执行周期的微指令首地址
		2. 执行周期：
		   $Ad(CMDR)\to CMAR$：每条微指令结束之后都需要进行
3. 确定微指令格式
4. 编写微指令码点
## 微程序设计分类
### 静态微程序设计
微程序无需改变，采用ROM
### 动态微程序设计
通过改变微指令和微程序改变机器指令
有利于仿真，采用EPROM
## 硬布线和微程序的比较

| 对比项目 | 微程序控制器                            | 硬布线控制器                            |

## 性能指标
### 吞吐率（TP）
单位时间内流水线完成的任务
一般取$\Delta t=一个时钟周期$
$$TP=\frac{n}{T_{k}}，T_{k}=(k+n-1)\Delta t$$
$$
	TP=\lim_{ n \to \infty } \frac{n}{(k+n-1)\Delta t}=\frac{1}{\Delta t} 
$$
### 加速比（S）
不使用流水线所用的时间与使用流水线所用的时间之比
$T_{0}=不使用流水线时的执行时间，T_{k}=使用流水线的执行时间$
$$
	S=\frac{T_{0}}{T_{k}}=\frac{kn\Delta t}{(k+n-1)\Delta t}=\frac{kn}{k+n-1}=\lim_{ n \to \infty } \frac{kn}{k+n-1}=k 
$$
### 效率（E）
流水线的设备利用率，即红色面积/蓝色面积
$$
	E=\frac{n个任务占用k时空区有效面积}{n个任务所用的时间与k个流水段所围成的时空区总面积}=\frac{T_{0}}{kT_{k}}=\lim_{ n \to \infty } \frac{T_{0}}{kT_{k}}=1 
$$

## 影响因素
### 结构相关（资源冲突）
多条指令在同一时刻争用统一资源而形成的冲突
**解决方案**：
1. 后一相关指令暂停一周期
2. 资源重复配置：
   资源存储器+指令存储器
### 数据相关（数据冲突）
必须等前一条指令执行完才能执行后一条指令的情况
![[Pasted image 20260620141712.png]]
**解决方案**：
1. 把遇到数据相关的指令及其后续指令都暂停一至几个时钟周期，直到数据相关问题消失后再继续执行。
	1. **硬件阻塞（stall）**
	2. **软件阻塞（NOP）**
2. 数据旁路技术
3. 编译优化：通过编译器调整指令顺序来解决数据相关
### 控制相关（控制冲突）
当流水线遇到转移指令和其他改变PC值的指令而造成断流时，会引起控制相关
![[Pasted image 20260620142157.png]]
**解决方案**：
1. 转移指令分支预测。
	1. 简单预测（永远猜true或者false）
	2. 动态预测（根据历史情况动态调整）
2. 预取转移成功和步成功两个控制流方向上的目标指令
3. 加快和提前形成条件码
4. 提高转移方向的猜准率

3. EX：运算，将结果存入EX段锁存器
4. M：空段
5. WB：将运算结果写回指定寄存器
### LOAD指令
![[Pasted image 20260620144855.png]]
1. IF：根据PC从指令Cache取指令到IF段的锁存器
2. ID：将基址寄存器的值放到锁存器A，将偏移量的值放到Imm
3. EX：运算，得到有效地址
4. M：从数据Cache中取数并放入锁存器
5. WB：将取出的数写回指定寄存器
### STORE指令
![[Pasted image 20260620145126.png]]
1. IF：根据PC从指令Cache取指令到IF段的锁存器
2. ID：将基址寄存器的值放到锁存器A，将偏移量的值放到Imm。将要存的数放到B
3. EX：运算，得到有效地址，并将锁存器B的内容放到锁存器Store
4. M：写入数据Cache
5. WB：空段
### 条件转移指令
![[Pasted image 20260620145549.png]]
1. IF：根据PC从指令Cache取指令到IF段的锁存器
2. ID：进行比较的两个数放入锁存器A、B；偏移量放入Imm
3. EX：运算，比较两个数
4. M：将目标PC值写回PC
5. WB：空段
### 无条件转移指令
![[Pasted image 20260620145701.png]]
1. IF：根据PC从指令Cache取指令到IF段的锁存器
2. ID：偏移量放入Imm
3. EX：将目标PC值写回PC
4. M：空段
5. WB：空段

| 对比项    | 细粒度多线程                   | 粗粒度多线程                               | 同时多线程（SMT）          |

| 线程切换代价 | 低                        | 高，需要重载流水线                            | NULL                |
| 并行性    | 指令级并行，线程间不并行             | 指令级并行，线程间不并行                         | 指令级并行，线程级并行         |
