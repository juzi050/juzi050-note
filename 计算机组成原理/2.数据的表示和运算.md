# 2.2.2 定点数的移位运算

通过改变各个数码位和小数点的相对位置，从而改变各数码位的位权。

## 逻辑移位（常用于处理无符号整数）

**逻辑左移**：高位移出丢弃，低位补0

**逻辑右移**：低位移出丢弃，高位补0

对于无符号整数，每逻辑左移一位，相当于$\times 2$

## 算术移位

**算数左移**：高位移出丢弃，低位补0。左移前后符号位不同，判断为溢出。

**逻辑右移**：低位移出丢弃，高位补符号位。

# 2.2.3.1 定点数的加减运算

## 例题：设机器字长为8位（含1位符号位），A=15,B=-24,求 $[A+B]_补和[A-B]_补$

$$A=+1111=[0,0001111]_原=[0,0001111]_补$$

$$B=-11000=[1,0011000]_原=[1,1101000]_补$$

$$[A+B]_补=[A]_补+[B]_补\\ =0,0001111+1,1101000\\ =1,1110111$$

$$[A-B]_补=[A]_补+[-B]_补\\ =0,0001111+0,0011000\\ =0,0100111$$

## 溢出判断

正+正才会上溢 -- 正+正=负

负+负才会下溢 -- 负+负=正

### 采用一位符号位，按照结果判断

$$A_s代表A的符号位，B_s代表B的符号位，运算结果的符号位为S_s。\\ 则溢出逻辑表达式为：V=A_S B_S\overline{S_S}+\overline{A_S}\ \overline{B_S}S_S$$

V=0->无溢出，V=1->有溢出。AB同号且S异号代表溢出。

### 采用一位符号位，根据数据位进位情况判断溢出

$$符号位进位C_S，最高数值位的进位C_1$$

$$V=C_S \oplus C_1$$

|     | $C_S$ | $C_1$ |
|-|-|-|
| 上溢  | 0     | 1     |
| 下溢  | 1     | 0     |

### 双符号位，正数符号为00，负数符号为11

最终结果01代表上溢出，10代表下溢

高位代表本应得到的符号位，低位代表实际得到的。

$$记两个符号位为S_{S1}S_{S2}，则V=S_{S1} \oplus S_{S2}$$

V=0代表无溢出，V=1代表有溢出

### 补充

双符号位补码又称：模4补码

单符号位补码又称：模2补码

双符号位补码在实际存储时只存储1个符号位，运算时会复制一个符号位。

# 2.2.3.2 无符号数的加减运算

## 加法

- 按位进行相加，往高位进位

## 减法

- 被减数不变，减数全部按位取反，末位+1，减法变加法
- 从最低位开始，按位相加，并往更高位进位

## 溢出判断

### 加法溢出

最高位的进位=1代表溢出，否则无溢出。

### 减法溢出

减法变加法，最高位产生的进位=0时，发生溢出，否则无溢出。

例如：以机器字长为8bit举例，A-B 变成 A+(-B)，计算机计算逻辑为 $A+B_补，其中B+B_补=2^8$，若 $进位=0，则代表A+B_补 < 2^8，则 A<B，发生减法溢出$

# 2.2.4.1 无符号整数的乘法运算原理

## 手算乘法

![[计算机组成原理-2.数据的表示和运算-10-fb52d3343d53.png]]

## 乘法原理

![[计算机组成原理-2.数据的表示和运算-11-2d0e02a189f9.png]]

初始化 $C_n=n$ ，$C=0$， $P=0000$

重复 n 轮加法、移位运算，直到计数器$\boldsymbol{C_n = 0}$ 

1. 将乘数寄存器Y的最低位，送入**控制逻辑**进行判断 
2. ①若Y的最低位为1，则执行加法，运算结果写回P（注意加法产生的进位需保存至进位触发器C）；  
②若Y的最低位为0，则什么也不做。 
3. 将【**C, P, Y**】视为整体，**逻辑右移**一位
4. 计数器$C_n$减1

## 溢出判断

两个 n bit **无符号整数相乘**，运算结果用 2n bit 暂存。通常仅保留低 n 位作为乘法结果。若**高 n 位不全为0**，说明发生溢出，此时可将 OF 标志位置为1

### 溢出处理

- 忽略溢出，会导致错误的运算结果
- 处理溢出：在乘法指令之后执行一条**溢出自陷指令**（例如：x86 的 INTO 指令（Interrupt on Overflow））。该指令会检查 OF 标志位，若OF==1，就会执行操作系统的”异常处理程序“

# 2.2.4.2 带符号整数的乘法原理（布斯乘法）

![[计算机组成原理-2.数据的表示和运算-12-feca156039ac.png]]

初始化 $P=0000,辅助位=0，C_n=n$

重复 n 轮**加/减法**、**移位**运算，直到计数器$\boldsymbol{C_n = 0}$：

1. 将乘数寄存器Y的最低位、辅助位，共**2bit**送入**控制逻辑**进行判断
2. 根据寄存器Y的最低位、辅助位，决定执行 **$+[X]_补$**、**$-[X]_补$** 还是 **$+0$**
3. 将【**P, Y, 辅助位**】视为整体，**算术右移**一位
4. 计数器$C_n$减1

| 寄存器Y最低位 | 辅助位 | 本轮操作 |
|-|-|-|
| 0 | 0 | +0 |
| 0 | 1 | $+[X]_补$ |
| 1 | 0 | $-[X]_补$ |
| 1 | 1 | +0 |

记忆技巧：

辅助位-Y最低位 = 1 ， $P+[X]_补$

辅助位-Y最低位 = 0 ， $P+0$

辅助位-Y最低为 = -1 ， $P-[X]_补$

## 溢出判断

两个 n bit **有符号整数相乘**，运算结果用 2n bit 暂存。通常仅保留低 n 位作为乘法结果。若**高 n+1 位不完全相同**，说明发生溢出，此时可将 OF 标志位置为1

判断思路如下：

之前讲过符号扩展，如果前 n+1 位都相同，则代表前面的都是符号扩展出来的，此时截断并不会影响结果。

### 溢出处理

- 忽略溢出，会导致错误的运算结果
- 处理溢出：在乘法指令之后执行一条**溢出自陷指令**（例如：x86 的 INTO 指令（Interrupt on Overflow））。该指令会检查 OF 标志位，若OF==1，就会执行操作系统的”异常处理程序“

## 阵列乘法器

![[计算机组成原理-2.数据的表示和运算-14-358623294df4.png]]

特点：可以在1个时钟内完成乘法

## 用逻辑运算、加/减运算等效实现乘法

```C++
/*用移位运算、加法运算等效实现32bit无符号数乘法*/
unsigned int multiply_unsigned(unsigned int x, unsigned int y) {
    unsigned int result = 0;
    for (int i = 0; i < 32; i++) {
        // 提取乘数 y 的最低位
        unsigned int bit = y & 1;
        // 如果当前位为 1，加上被乘数 x
        if (bit) {
            result += x;
        }
        // 被乘数 x 左移一位，乘数 y 右移一位
        x <<= 1;
        y >>= 1;
    }
    return result;
}
```

**优点** ：在不支持乘法指令的计算机中，可以等效实现乘法效果

**缺点** ：运算速度很慢

# 2.2.4.4 无符号整数除法原理

$$被除数 = 商 \times 除数 + 余数$$

## 手算

![[计算机组成原理-2.数据的表示和运算-15-3117c8ee1211.png]]

如果 $中间余数 \ge 除数$则上商1，否则商0

## 除法原理

![[计算机组成原理-2.数据的表示和运算-16-29a7a2012383.png]]

该除法器支持 $2n\ bit \div n bit$（双精度除法）和$n\ bit \div nbit$（单精度除法），最终得到 n bit 商，n bit 余数。

初始化 $Y=除数$，$被除数放入[R,Q]$，$C_n=n$。

特殊情况检查：

- 除数为0，发生除数为0异常，停止除法运算，调出操作系统的异常处理程序
- 如果 $|被除数| <|除数|$，则商=0，余数=被除数，除法器不必再执行

进行 n+1 轮处理，上商规则：如果 $[R]-R[Y] \ge 0$，则上商1，否则上商0

1. 第一轮特殊处理（商溢出判断）

   1. 直接上商，若第一位商=1，发生”商溢出“异常，停止除法运算
   2. 直接上商，若第一位商=0，说明不会发生”商溢出“，不必保存这位商，也不让 $C_n \texttt{--}$，除法继续
2. 其余 n 轮处理

   1. 先左移，空出的位用于上商
   2. 上商，背后的过程可能进行 加法/减法
   3. 计数器 $C_n \texttt{--}$，当计数器 $C_n=0$时，除法运算结束
3. 结束 R 保存余数，Q 保存商

## 概念补充

商溢出异常和除0异常都被叫做除法异常。

# 2.2.4.5 带符号整数除法

![[计算机组成原理-2.数据的表示和运算-17-0e46cea05924.png]]

支持$2n \ bit \div n \ bit$，最终得到n bit商、n bit 余数。

初始化：

被除数通过符号扩展为 2n bit，放入寄存器[R,Q]

除数放到Y，$C_n=n$

n轮处理

1. 先左移，空出的位用于上商
2. 上商

   1. 由控制逻辑根据中间余数和除数的符号组合，决定本轮左减法/加法（**同号减，异号加**）
   2. 根据ALU运算结果（新余数）的符号位决定上商为1还是为0
   3. 新余数与老余数相比，符号不变->上商1，符号变->上商0
3. 计数器$C_n \texttt{--}$，当计数器$C_n=0$时，除法运算结束
4. 当被除数和除数异号的时候，商Q还要取补（全部位取反，末位+1）才是最终答案。

## 特殊情况

对于补码的单精度除法（$n \ bit \div n \ bit$，得到n bit 商，n bit 余数），仅$绝对值最大的负数\div -1$才可能发生商溢出。

- **上溢（Overflow）**：运算结果超出最大规格化正数或者小于绝对值最大的规格化负数。

  - 处理方式：  
  将运算结果设为$+\infty$或者$-\infty$  
  设置浮点数上溢异常标志位（例如：x86会将浮点数运算单元FPU的OE标志位 Overflow Exception 置为1）
- **下溢（Undeflow）**：运算结果小于最小规格化正数或者大于绝对值最大的规格化负数

  - 处理方式：  
  结果落入非规格化区间->用**非规格化浮点数**存储；若结果太小（真值逼近于0）->按**机器零**存储  
  若下溢到**机器零**，设置浮点数下溢异常标志位（例如：x86会将浮点数运算单元FPU的UE标志位 Underflow Exception 置为1）

## 就近舍入

IEEE默认舍入模式为就近舍入，类似”0舍1入“。如果3个舍弃位刚好是100，则若1+23bit的**末位为0**，**直接截断多余位**即可，若1+23bit尾数的**末位为1**，则**截断多余位，并在”末位+1“**

1. 上溢

   1. 浮点数的溢出时以**阶码**进行判断的，阶码全1说明**上溢**
2. 下溢

   1. 阶码全0

## 边界对齐

现代计算机通常是按字节编址，即每个字节对应一个地址。

通常也支持按字/半字/字节寻址。

假设存储字长为32位，则一个字=32bit，半字=16bit。每次访存只能读/写一个字
