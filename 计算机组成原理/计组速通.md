# 第 1 章 概论
## 计算机硬件系统（五大部件）
完整知识：[计算机硬件的基本组成](计算机组成原理/1.2.1+1.2.2计算机硬件的基本组成)

原理：冯·诺依曼计算机硬件由五大部件组成，现代计算机通常理解成“以存储器为中心”，CPU 负责取指令、分析指令、执行指令。

| 部件 | 功能 |
| --- | --- |
| 运算器 | 在控制器控制下，进行算术运算和逻辑运算，直接影响运算速度 |
| 存储器 | 存放指令和数据；基本操作是读出、写入；每个存储单元有唯一地址 |
| 控制器 | 对当前指令译码，产生控制信号，使全机自动、协调工作 |
| 输入设备 | 把外界信息转换为计算机能识别的二进制代码 |
| 输出设备 | 把计算机处理结果转换成人或外设能接收的形式 |

**速记**：CPU = 运算器 + 控制器；主机 = CPU + 主存；完整计算机系统 = 硬件系统 + 软件系统。

## 冯·诺依曼体系结构特点
原理：冯·诺依曼体系结构的核心是**二进制 + 存储程序控制**。

| 特点 | 说明 |
| --- | --- |
| 二进制表示 | 数据和指令都用二进制表示 |
| 指令结构 | 指令由操作码和地址码组成 |
| 存储程序 | 程序和原始数据预先存入主存，机器自动取指执行 |
| 顺序执行 | 一般按指令在存储器中的顺序执行，分支由转移指令实现 |
| 五大部件 | 运算器、控制器、存储器、输入设备、输出设备 |
| 早期结构 | 以运算器为中心，I/O 与存储器交换数据要经过运算器 |
| 现代结构 | 更强调以存储器为中心，减少数据绕行 |

## 计算机三种语言
原理：计算机语言分为机器语言、汇编语言和高级语言，它们对应从硬件到用户的不同抽象层。

| 语言 | 构成 | 与硬件关系 | 能否直接执行 | 可读性 | 执行效率 |
| --- | --- | --- | --- | --- | --- |
| 机器语言 | 0/1 代码 | 面向机器 | 可以 | 低 | 高 |
| 汇编语言 | 助记符 | 面向机器 | 不可以，需汇编 | 较低 | 高 |
| 高级语言 | 类自然语言/数学表达 | 面向用户 | 不可以，需编译或解释 | 高 | 相对低 |

例子套路：C 程序不会直接被 CPU 执行，要先变成汇编，再变成机器码。

```text
C源程序 → 预处理 → 编译 → 汇编 → 链接 → 可执行机器程序 → CPU执行
```

例如数组求和：C 语言的 `sum += a[i]` 会被编译成 RISC-V 中的移位求地址、访存取数、加法、条件分支等机器指令。
## 计算机系统层次结构
原理：计算机系统按抽象层次自底向上组织，本课程主要看“微体系结构层”和“机器指令层”。

| 层次 | 说明 |
| --- | --- |
| 逻辑实现层 | 门电路、触发器等数字逻辑实现 |
| 微体系结构层 | CPU 内部数据通路、控制器、寄存器、ALU 等 |
| 机器指令层 | 指令系统 ISA，机器指令格式与执行 |
| 操作系统层 | 管理软硬件资源，提供系统调用 |
| 语言处理程序及系统软件层 | 编译器、汇编器、链接器、数据库、网络管理等 |
| 应用软件层 | 用户直接使用的应用程序 |

**考试速记**：硬件向上提供机器指令接口，软件最终都要落到机器指令上执行。
## 计算机性能指标计算（可与 7.1.4 节结合，期末考试）
原理：评价 CPU 性能不能只看主频，要综合指令条数、CPI 和时钟周期。

| 指标       | 公式/说明                                                |
| -------- | ---------------------------------------------------- |
| 主频       | $f$，时钟周期 $T=1/f$                                     |
| CPI      | 每条指令平均需要的时钟周期数                                       |
| CPU 执行时间 | $T_{CPU}=I\times CPI\times T=\dfrac{I\times CPI}{f}$ |
| 加权平均 CPI | $\overline{CPI}=\sum w_i\times CPI_i$                |
| MIPS     | $MIPS=\dfrac{f}{\overline{CPI}\times 10^6}$          |
| 存储容量     | 存储单元个数 $\times$ 存储字长                                 |

**例 1.1：CPI、MIPS 与程序执行时间**

题设：主频 `1GHz`，4 类指令如下。

| 指令类型 | CPI | 出现比例 |
| --- | ---: | ---: |
| 运算及寄存器传送 | 1 | 55% |
| 分支 | 4 | 20% |
| 访存命中 Cache | 3 | 18% |
| 访存未命中 Cache | 10 | 7% |

1. 平均 CPI：
   $$\overline{CPI}=1\times0.55+4\times0.2+3\times0.18+10\times0.07=2.59$$
2. 主频 `1GHz`，时钟周期 $T=1ns$，平均每条指令时间 $2.59ns$。
3. 运算速度：
   $$MIPS=\frac{1}{2.59ns\times10^6}\approx386$$
4. 若程序 4 类指令条数分别为 `20、1、4、1`：
   $$T=(20\times1+1\times4+4\times3+1\times10)\times1ns=46ns$$
5. 若新架构主频升到 `1.5GHz`，但每类 CPI 都加 1：
   - 加权 CPI：$3.59$
   - 平均指令时间：$3.59/1.5\approx2.39ns$
   - MIPS：约 `418`，只看 MIPS 更好。
   - 但对上面那段程序：
     $$T=(20\times2+1\times5+4\times4+1\times11)\times\frac{1}{1.5}ns=48ns$$
     比原来的 `46ns` 慢。

**结论**：性能评价要看具体程序，不能只看主频、CPI 或 MIPS；优化高频事件收益最大。

**例 1.2：不同机器性能比较**

| 指令类型 | A | B | C | D |
| --- | ---: | ---: | ---: | ---: |
| M1 的 CPI | 1 | 3 | 4 | 3 |
| M1 指令条数/百万 | 8 | 4 | 2 | 4 |
| M2 的 CPI | 1 | 2 | 4 | 3 |
| M2 指令条数/百万 | 10 | 8 | 2 | 4 |

1. 平均 CPI：$CPI_{M1}=20/9\approx2.22$，$CPI_{M2}=23/12\approx1.92$，只看 CPI 是 M2 好。
2. 若主频相同：$T_{M1}:T_{M2}=40:46$，实际 M1 更快。
3. 若 $f_{M1}=1.2GHz$ 且两者运行时间相同：
   $$40t_{M1}=46t_{M2}\Rightarrow 46f_{M1}=40f_{M2}\Rightarrow f_{M2}\approx1.38GHz$$

# 第 3 章 信息编码与数据表示
## 定点数表示（原码、补码、移码）  
原理：机器数是数值在机器中的编码，真值是数学意义上的值。定点数的小数点位置固定，可分为定点整数和定点小数。

| 编码 | 正数 | 负数 | 0 的表示 | 整数表示范围（1 位符号 + n 位数值） | 常用场景 |
| --- | --- | --- | --- | --- | --- |
| 原码 | 符号位 0，数值位不变 | 符号位 1，数值位不变 | +0、-0 | $-(2^n-1)$ 到 $+(2^n-1)$ | 直观表示、浮点尾数符号 |
| 反码 | 同原码 | 符号位不变，数值位取反 | +0、-0 | 同原码 | 理解补码的过渡 |
| 补码 | 同原码 | 反码末位 +1 | 唯一 0 | $-2^n$ 到 $2^n-1$ | 整数运算核心 |
| 移码 | 补码符号位取反 | 补码符号位取反 | 唯一 0 | 同补码 | 浮点阶码 |

**速记**：

1. 负数补码 = 原码数值位取反再加 1。
2. 移码 = 补码符号位取反。
3. 计算机中的 `int` 通常用补码表示。

**例：写机器数**

| 真值 | 原码 |
| --- | --- |
| $+0.1001$ | `0.1001` |
| $-0.1001$ | `1.1001` |
| $+1110$ | `0,1110` |
| $-1110$ | `1,1110` |

**例：移码**

| 真值 | 移码 |
| --- | --- |
| $+1110$ | `1,1110` |
| $-1110$ | `0,0010` |

**例：已知机器数求真值**

| 机器数 | 真值 |
| --- | --- |
| 8 位 `$[X]_原=1,0000000$` | $-0$ |
| 8 位 `$[X]_补=1,0000000$` | $-128$ |
| 8 位 `$[X]_移=1,0000000$` | $0$ |

## 浮点数规格化表示（自定义格式、IEEE754 标准）  
原理：浮点数用 $N=M\times2^E$ 表示，阶码 $E$ 决定范围，尾数 $M$ 决定精度。

| 字段 | 含义 | 常见编码 |
| --- | --- | --- |
| 阶码 E | 指数，决定小数点位置 | 移码 |
| 尾数 M | 有效数字，决定精度 | 原码或补码 |

**规格化判断**：

| 尾数编码 | 规格化条件 |
| --- | --- |
| 原码尾数 | 最高有效位为 1，即没有前导 0 |
| 补码尾数 | 符号位和最高数值位不同，即 $M_s\oplus M_1=1$ |

**例 3.8：自定义浮点数格式**

格式：16 位浮点数，阶码 6 位（含阶符）用移码，尾数 10 位（含数符）用补码，阶码在前。

求 $X=-27/128$ 的规格化表示：

1. $X=(-0.0011011)_2$
2. 规格化：$X=-0.11011\times2^{-2}$
3. 阶码：$E_X=-00010$，$[E_X]_移=0,11110$
4. 尾数：$M_X=-0.110110000$，$[M_X]_补=1.001010000$
5. 拼接：
   $$[X]_浮=0,11110\ 1.001010000=7A50H$$

已知 $[Y]_浮=99BEH$ 求真值：

1. 二进制：`1001 1001 1011 1110`
2. 拆分：$[E_Y]_移=1,00110$，$[M_Y]_补=0.110111110$
3. 得：$E_Y=+00110$，$M_Y=+0.110111110$
4. 真值：
   $$Y=0.110111110\times2^6=110111.11_2=55.75$$

**IEEE754 单精度**：

| 字段 | 位数 | 含义 |
| --- | ---: | --- |
| S | 1 | 数符，0 正 1 负 |
| E | 8 | 阶码，偏置值 Bias=127 |
| fraction | 23 | 尾数小数部分，隐藏最高位 1 |

规格化数真值：

$$(-1)^S\times(1.fraction)_2\times2^{E-127}$$

**例 3.10：IEEE754 解码**

`41360000H = 0 10000010 01101100000000000000000`

1. $S=0$，正数。
2. $E=130$，真阶码 $E'=130-127=3$。
3. 尾数为 $1.011011_2$。
4. 真值：$1.011011_2\times2^3=11.375$。

**练习：`float x=0.1`**

单精度中 `0.1` 实际约为：

$$0.100000001490116119384765625$$

所以它比数学上的 `0.1` 略大。考点：二进制浮点无法精确表示很多十进制小数。
## 计算机高级语言中数据类型转换涉及的编码（考试会考）
原理：高级语言的数据类型转换本质上是“同一串二进制位按不同编码解释”，或“位数改变时做扩展/截断”。

| 转换 | 规则 | 易错点 |
| --- | --- | --- |
| 短有符号整数 → 长有符号整数 | 符号扩展 | 最高位是 1 时高位补 1 |
| 短无符号整数 → 长整数 | 零扩展 | 高位补 0 |
| 长整数 → 短整数 | 截断低位 | 可能改变真值 |
| 有符号 ↔ 无符号 | 位模式不变，解释方式变 | 负数会变成很大的无符号数 |
| 整数 ↔ 浮点 | 可能舍入或溢出 | 精度不一定保留 |

**大小端**：

| 模式 | 规则 | 例：`234589ABH` 从低地址开始存 |
| --- | --- | --- |
| 小端 | 低有效字节放低地址 | `AB 89 45 23` |
| 大端 | 高有效字节放低地址 | `23 45 89 AB` |

**例 3.13：强制类型转换**

| 语句 | 十进制值 | 十六进制编码 | 分析 |
| --- | ---: | --- | --- |
| `short si=-1234;` | -1234 | `FB2E` | 16 位补码 |
| `int i=si;` | -1234 | `FFFF FB2E` | 有符号短数转长数，符号扩展 |
| `unsigned short usi=si;` | 64302 | `FB2E` | 位模式不变，按无符号解释 |

**考试分析套路**：先确定变量位数，再写补码/IEEE754 编码，最后按目标类型解释。
# 第 4 章 运算方法与运算器
## 补码加减运算、移位运算
原理：补码的最大作用是让符号位和数值位一起参加运算，并把减法转化成加法。

$$[X+Y]_补=[X]_补+[Y]_补$$

$$[X-Y]_补=[X]_补+[-Y]_补$$

| 溢出判断方法 | 判断规则 |
| --- | --- |
| 单符号位 | 同号相加结果异号；正数减负数结果为负；负数减正数结果为正 |
| 进位判溢 | 最高数值位进位 $C_1$ 与符号位进位 $C_s$ 不同则溢出，$V=C_1\oplus C_s$ |
| 双符号位 | 结果两个符号位不同则溢出，`01` 正溢出，`10` 负溢出 |

**例 4.2：补码加减**

已知 $X=+1011$，$Y=-0100$：

1. 写补码：$[X]_补=0,1011$，$[Y]_补=1,1100$，$[-Y]_补=0,0100$。
2. $[X+Y]_补=0,0111$，即 $+7$。
3. $[X-Y]_补=0,1111$，即 $+15$。

**例 4.4：判溢**

1. $X=+1000$，$Y=+1001$，计算 $X+Y$：正数加正数却得到负数，正溢出；进位法表现为 $C_1\neq C_s$。
2. $X=-1000$，$Y=+1001$，用双符号位计算 $X-Y$：若结果符号位为 `10`，表示负溢出。

**移位运算**：

| 移位 | 对象 | 空位补什么 | 用途/风险 |
| --- | --- | --- | --- |
| 逻辑左移 | 位串/无符号数 | 低位补 0 | 相当于乘 2，可能丢高位 |
| 逻辑右移 | 无符号数 | 高位补 0 | 相当于无符号除 2 |
| 算术左移 | 补码有符号数 | 低位补 0 | 符号位可能改变，注意溢出 |
| 算术右移 | 补码有符号数 | 高位补符号位 | 保持正负号 |
| 循环移位 | 位串 | 移出位回填 | 常和 CF 配合 |

**例 4.1 易错点**：C 语言 `<<` 不检查左移溢出；补码左移前若符号位与最高数值位不同，左移后会溢出。
## 运算器的结构、数据通路
原理：运算器核心是 ALU，配合通用寄存器堆、暂存器、内部总线、标志寄存器完成数据处理。

| 结构 | 特点 |
| --- | --- |
| 单总线结构 | 结构简单，但一个 CPU 周期内总线上只能有一个数据 |
| 双总线/多总线结构 | 可同时读多个操作数，周期数更少 |
| 直接连接结构 | 部件间用专用线路连接，效率高，适合流水线 CPU |

**例 4.10：单总线执行 `ADD R3,R2,R7`**

功能：`R2+R7→R3`。

| CPU 周期 | 微操作 | 控制信号示意 |
| --- | --- | --- |
| 1 | `R2→LA` | `RA=2, GRout=1, LAin=1` |
| 2 | `R7→LB` | `RA=7, GRout=1, LBin=1` |
| 3 | `ALU执行 A+B，F→R3` | `ALU_OP=加法, ALUout=1, GRin=1, WA=3` |

结论：单总线需要 3 个 CPU 周期。

**例 4.11：双总线执行同一指令**

| CPU 周期 | 微操作 |
| --- | --- |
| 1 | `R2→IB1, R7→IB2, F=A+B, F→LC` |
| 2 | `LC→R3` |

结论：双总线需要 2 个 CPU 周期。分析原则：同一个 CPU 周期内，某条总线上的数据必须唯一，且不能保留到下一个周期。
## 标志寄存器
原理：标志寄存器保存 ALU 运算结果状态，常作为条件转移指令的判断依据。

| 标志 | 含义 | 置位条件 |
| --- | --- | --- |
| ZF | 零标志 | 结果为 0 |
| SF | 符号标志 | 结果最高位为 1 |
| CF | 进位/借位标志 | 无符号加法进位或减法借位 |
| OF | 溢出标志 | 有符号运算溢出 |
| PF | 奇偶标志 | 低字节中 1 的个数为偶数 |

**例 4.12：`80H + 80H`**

```asm
MOV AL,80H
ADD AL,80H
```

结果：`AL=00H`。

| 标志 | 值 | 分析 |
| --- | ---: | --- |
| ZF | 1 | 结果为 0 |
| CF | 1 | 无符号加法产生进位 |
| OF | 1 | 若按有符号数，`-128 + -128` 超范围 |
| SF | 0 | 结果最高位为 0 |
| PF | 1 | `00H` 中 1 的个数为偶数 |

**判断题速记**：`JC/JB` 看 CF，常用于无符号比较；有符号比较更多看 SF、OF、ZF 组合。
## 浮点数的加减法运算
原理：浮点加减不能直接加尾数，必须先对阶。

1. 对阶：小阶向大阶看齐，尾数右移。
2. 尾数加减：按尾数编码规则运算。
3. 规格化：左规或右规，并调整阶码。
4. 舍入：按题目要求，如 0 舍 1 入。
5. 判溢：阶码上溢/下溢。

**例 4.14：浮点加法套路**

格式：12 位浮点数，阶码 4 位（含阶符）、尾数 8 位（含数符），均用补码。

$$X=-0.1001011\times2^{001},\quad Y=0.1100101\times2^{-010}$$

1. 按格式写出：
   - $[X]_浮=0,001\ 1.0110101$
   - $[Y]_浮=1,110\ 0.1100101$
2. 对阶：$\Delta E=E_X-E_Y=3$，所以 $Y$ 的尾数右移 3 位。
3. 尾数相加。
4. 结果规格化。
5. 按 0 舍 1 入法舍入。
6. 检查阶码是否上溢或下溢。

**考试易错**：对阶永远是“小阶向大阶看齐”，因为尾数右移只损失精度，若大阶向小阶看齐会让数值放大出错。
# 第 5 章 存储系统
## 存储器的分类、层次结构
原理：存储体系用“容量、速度、价格”的矛盾来分层，越靠近 CPU 越快、越小、越贵。

| 层次 | 特点 |
| --- | --- |
| 寄存器 | 最快、容量最小、价格最高 |
| Cache | 缓解 CPU 与主存速度差 |
| 主存 | CPU 可直接访问，容量较大 |
| 外存 | 容量最大、速度慢、非易失 |

| 存储器 | 特点 | 常用位置 |
| --- | --- | --- |
| SRAM | 快、不需刷新、贵 | Cache |
| DRAM | 集成度高、便宜、需刷新 | 主存 |
| ROM | 只读/非易失 | 固件、系统程序区 |
| Flash | 可擦写非易失 | U 盘、SSD |

**局部性原理**：时间局部性是“刚访问过的还可能再访问”，空间局部性是“访问某地址后，附近地址也可能被访问”。Cache 和虚拟存储器都依赖局部性。
## 多字节数据在存储器中的存放顺序：大小端
原理：大小端只影响**多字节数据在内存中的字节顺序**，不改变寄存器里这个数的数值。

| 模式 | 规则 | 例：`234589ABH` 从低地址开始存 |
| --- | --- | --- |
| 小端 | 低有效字节放低地址 | `AB 89 45 23` |
| 大端 | 高有效字节放低地址 | `23 45 89 AB` |

**考试套路**：

1. 先把数据按字节拆开。
2. 小端：最低字节先放；大端：最高字节先放。
3. `lb/lh/lw` 取数后还要看是符号扩展还是零扩展。
## 主存储器 SRAM、ROM 的扩展设计
原理：芯片容量不够时，用位扩展、字扩展、字位扩展组成目标主存。

| 扩展方式 | 解决问题 | 做法 |
| --- | --- | --- |
| 位扩展 | 每个存储单元位数不够 | 多片并联，地址线和片选相同，数据线拼宽 |
| 字扩展 | 存储单元个数不够 | 数据线并联，高位地址经译码器产生片选 |
| 字位扩展 | 字数和位数都不够 | 先位扩展成一组，再多组字扩展 |

连接套路：

1. 低位地址线接芯片片内地址。
2. 高位地址线进译码器，译码输出接片选。
3. 数据线按位宽连接。
4. `MREQ#` 常接译码器使能端。
5. SRAM/RAM 接读写控制信号；ROM 不接写信号。

**例 5-2：`256K×8` SRAM 构成 `2048K×32`**

1. 芯片数：
   $$\frac{2048K}{256K}\times\frac{32}{8}=8\times4=32\text{片}$$
2. 每 4 片位扩展成一组 `256K×32`。
3. 共 8 组做字扩展。
4. `2048K=2M=2^21`，所以存储器需要 21 根地址线。
5. 低 18 位接芯片地址线，高 3 位接 `3:8` 译码器产生 8 组片选。
6. `MREQ#` 作为译码器使能。

## 多体交叉存储器
原理：多体交叉存储器把主存分成多个独立存储体，让连续地址分散到不同体中，从而流水式并行访问。

| 编址 | 地址分配 | 特点 |
| --- | --- | --- |
| 顺序编址/高位交叉 | 高位选体，低位选体内单元 | 一个体内地址连续，适合扩容 |
| 交叉编址/低位交叉 | 低位选体，高位选体内单元 | 连续地址落到不同体，适合连续高速访问 |

常用公式：若存储周期为 $T$，总线传送周期为 $\tau$，连续读 $n$ 个字：

$$t=T+(n-1)\tau$$

为不断流，体数通常满足：

$$m\ge \frac{T}{\tau}$$

## Cache 
原理：Cache 利用程序局部性，把近期可能访问的数据块放到 CPU 附近。CPU 访问时先查 Cache，命中则直接取，未命中则访问主存并调块。

| 概念 | 含义 |
| --- | --- |
| 块/行 | Cache 与主存交换的基本单位 |
| 命中 | 访问的数据在 Cache 中 |
| 未命中 | 数据不在 Cache 中，要访问主存 |
| 标记 Tag | 判断 Cache 中的块来自主存哪个位置 |

**例：主存 4GB、块大小 16B、访问地址 `234589ABH`**

| 映射方式 | Cache 条件 | 块内地址 | 行/组地址 | 标记 |
| --- | --- | ---: | ---: | ---: |
| 直接映射 | Cache `512KB` | 4 位：`BH` | 15 位：`589AH` | 13 位：`468H` |
| 全相联 | Cache `8KB` | 4 位：`BH` | 无 | 28 位：`234589AH` |
| 4 路组相联 | Cache `512KB` | 4 位：`BH` | 13 位：`189AH` | 15 位：`11A2H` |

计算套路：块大小 $16B=2^4B$，所以块内地址 4 位；行数 = Cache 容量 / 块大小；组数 = 行数 / 路数；标记位 = 主存地址位数 - 组/行位数 - 块内位数。
### 命中率
$t_{a}=ht_{c}+(1-h)\times(t_{m}+t_{c})$代表串行访问：先给cache发出读信号，读不命中的情况下向主存发出读信号
$t_{a}=ht_{c}+(1-h)\times t_{m}$代表并行访问：同步向cache和主存发出读信号
![[Pasted image 20260628191938.png]]

原理：命中率 $h$ 越高，平均访问时间越接近 Cache 访问时间 $t_c$。串行访问会在未命中时多花一次 Cache 查询时间；并行访问硬件更复杂，但未命中代价表达式更小。

# 第 6 章 指令系统
## 指令格式的分析与设计（包括指令操作码扩展技术） 
原理：机器指令是 CPU 能识别执行的 0/1 代码串，通常由操作码字段和地址码字段组成。

| 字段 | 作用 |
| --- | --- |
| 操作码 OP | 指明执行什么操作 |
| 地址码 A | 指明操作数、操作数地址或寻址信息 |

| 地址格式 | 功能形式 | 说明 |
| --- | --- | --- |
| 三地址 | `(A1) OP (A2) → A3` | 两个源操作数，一个目的操作数 |
| 二地址 | `(A1) OP (A2) → A1` | 目的操作数兼源操作数 |
| 一地址 | `(ACC) OP (A) → ACC` | 一个操作数隐含在累加器 |
| 零地址 | 无显式操作数 | 如 `NOP`、`HLT` 或堆栈指令 |

**操作码扩展技术**：把不用的地址码字段扩展为操作码，让不同地址数的指令具有不同长度的操作码。

| 优点 | 缺点 |
| --- | --- |
| 缩短平均指令长度；增加可表示操作种类 | 指令译码更复杂，控制器设计难度增加 |

考试套路：先看固定指令字长，再统计每类指令占用的操作码前缀，剩余编码能否继续扩展。
## 指令的寻址方式（指令寻址和数据寻址） 
原理：寻址方式解决“下一条指令在哪里”和“本条指令的操作数在哪里”。

| 类型 | 说明 |
| --- | --- |
| 顺序寻址 | PC 自动加上当前指令长度，取下一条指令 |
| 跳跃寻址 | 执行转移指令，把目标地址送 PC |

| 数据寻址方式 | 有效地址/操作数 | 速记 |
| --- | --- | --- |
| 立即寻址 | `Operand=A` | 操作数就在指令中 |
| 直接寻址 | `EA=A`，`Operand=Mem[A]` | 指令中给地址 |
| 间接寻址 | `EA=Mem[A]`，`Operand=Mem[EA]` | 地址的地址 |
| 寄存器寻址 | `Operand=Reg[Ri]` | 操作数在寄存器 |
| 寄存器间接 | `EA=Reg[Ri]`，`Operand=Mem[EA]` | 地址在寄存器 |
| 变址寻址 | `EA=RI+A` | 数组、字符串、向量 |
| 基址寻址 | `EA=Rb+A` | 程序重定位、扩大寻址范围 |
| 相对寻址 | `EA=PC+A` | 转移指令常用 |
| 堆栈寻址 | `EA=SP` | 中断和子程序现场 |

**例 6.3：变址寻址分析**

x86 程序用变址寄存器遍历 `num` 数组的 10 个 16 位数据。

1. 程序功能：把数组 `num` 的 10 个数据累加，累加和在 `AX` 中。
2. 每累加一个 16 位数据，变址寄存器 `ESI` 加 2。
3. 因为 16 位数据占 2 个地址，所以 x86 是按字节编址。
4. 对应 C 语言就是固定次数 `for` 循环累加数组。
## 针对特定指令系统：机器指令的编码和解码、执行结果（汇编语言与机器语言之间的转换） 
原理：机器指令解码就是反汇编过程，先看 opcode 判断格式，再拆字段判断寄存器、立即数和功能码。

RV32I 常见格式：

| 格式 | 用途 | 功能 |
| --- | --- | --- |
| R 型 | 寄存器运算 | `Reg[rs1] op Reg[rs2] → Reg[rd]` |
| I 型 | 立即数运算、load、jalr | `Reg[rs1] op imm → Reg[rd]` |
| S 型 | store | `Reg[rs2] → Mem[rs1+imm]` |
| B 型 | 条件分支 | 满足条件则 `PC+offset→PC` |
| U 型 | 大立即数 | `lui/auipc` |
| J 型 | 无条件跳转 | `jal` |

**例 6.6：load/store 与小端**

题设：地址 `401280H` 的 32 位数据为 `234589ABH`，小端模式。

```asm
lui  t0,0x401       # t0=0x401000
addi t0,t0,0x200    # t0=0x401200
lw   t1,0x80(t0)    # t1=234589ABH
lh   t2,0x80(t0)    # t2=FFFF89ABH，符号扩展
lhu  t3,0x80(t0)    # t3=000089ABH，零扩展
lh   t4,0x82(t0)    # t4=00002345H
lb   t5,0x80(t0)    # t5=FFFFFFABH
lb   t6,0x83(t0)    # t6=00000023H
sw   t4,0x80(t0)
sh   t3,0x82(t0)
sb   t5,0x83(t0)
```

最终 结果：主存 `401280H` 开始的 4B 内容为 `ABAB2345H`。

**例 6.10：机器码解码套路**

1. 从 PC 指向的主存单元取 32 位机器码。
2. 写成二进制，取低 7 位 opcode。
3. 根据 opcode 判断 R/I/S/B/U/J 格式。
4. 拆 `rd、rs1、rs2、funct3、funct7、imm`。
5. 按指令功能更新寄存器、存储器或 PC。
## 针对 RISC-V 或者 ARM：从高级语言程序到机器语言的转换
原理：高级语言语句会被编译成寄存器传送、运算、访存、转移等基本指令序列。

**例 6.5：C 到 RV32I**

```c
int a,b,c,w;
a = -3;
b = 1000;
c = 7000;
w = 8*a + b - c;
```

假设 `a,b,c,w` 分配到 `x5~x8`：

```asm
addi x5,x0,-3       # a=-3
addi x6,x0,1000     # b=1000
lui  x7,2           # 处理 7000=0x1B58
addi x7,x7,0xB58    # c=7000
slli x8,x5,3        # 8a
add  x8,x8,x6       # 8a+b
sub  x8,x8,x7       # 8a+b-c
```

易错点：`addi` 的 12 位立即数会符号扩展，`0xB58` 最高位为 1，会被当负数，所以 `lui` 的立即数要按 `imm32[31:12]+imm32[11]` 修正。

**例 6.9：while 循环翻译**

```c
while (data[i] != y)
    i += 1;
```

寄存器约定：`data` 首地址在 `s4(x20)`，`i` 在 `s9(x25)`，`y` 在 `s10(x26)`。

```asm
Loop:
    slli t1,s9,2       # t1=i*4
    add  t1,t1,s4      # t1=&data[i]
    lw   t2,0(t1)      # t2=data[i]
    beq  t2,s10,exit   # 若 data[i]==y，退出
    addi s9,s9,1       # i++
    jal  x0,Loop       # 无条件回到循环入口
exit:
```

考试套路：数组元素地址 = 基址 + 元素大小 × 下标；循环条件通常用条件分支；回到循环入口可用 `jal x0,Loop`。
# 第 7 章 控制器
## 控制器的功能和组成  
原理：控制器是全机控制中心，根据机器指令产生执行所需的操作控制信号，协调各部件工作。

| 部件 | 功能 |
| --- | --- |
| PC | 存放下一条或当前相关指令地址 |
| IR | 存放当前指令码 |
| AR | 存放访问主存或 I/O 的地址 |
| DR | 存放访问主存或 I/O 的数据 |
| 指令译码器 ID | 对 IR 中操作码译码，识别指令 |
| 时序系统 | 产生机器周期、节拍、工作脉冲 |
| 控制单元 CU | 产生取指、译码、执行所需控制信号序列 |

控制器功能：取指令、分析指令、执行指令、中断处理、响应特殊请求。

时序概念：

| 概念 | 含义 |
| --- | --- |
| 指令周期 | 取出并执行完一条指令的时间 |
| 机器周期/CPU 周期 | 指令执行的一个阶段 |
| 时钟周期/节拍 | CPU 主频脉冲周期 |

**例 7.1**：主频 `100MHz`，平均每条指令 3 个机器周期，每周期 2 个节拍，则 $CPI=3\times2=6$，MIPS 约 $100/6=16.7$。若平均每条指令访存 1.5 次且每次插 1 个等待节拍，则 $CPI'=7.5$，平均指令时间 $75ns$。
## 硬布线控制器的原理与设计  
原理：硬布线控制器用组合逻辑产生控制信号。

```text
输入：指令译码信号 + 时序信号 + 状态条件
输出：各微操作控制信号
```

| 特点 | 说明 |
| --- | --- |
| 优点 | 速度快，适合 RISC |
| 缺点 | 设计复杂，修改和扩展困难 |

设计步骤：

1. 分析指令格式和功能。
2. 确定数据通路和所需部件。
3. 写出每条指令每个机器周期的微操作。
4. 列出每个控制信号的产生条件。
5. 化简逻辑表达式并实现电路。

与微程序控制器对比：硬布线像“电路直接判定”，微程序像“从控制存储器逐条取微指令解释执行”。
## 单周期、多周期与流水线实现的性能对比  
原理：三种 CPU 实现方式主要区别在“一个时钟周期做多少事”和“多条指令能否重叠”。

| 实现 | 特点 | 优缺点 |
| --- | --- | --- |
| 单周期 | 每条指令一个时钟周期完成 | 控制简单，但周期要按最慢指令设计 |
| 多周期 | 一条指令拆成多个周期 | 周期可按最长阶段设计，不同指令周期数不同 |
| 流水线 | 多条指令分阶段重叠执行 | 吞吐率高，但要处理相关和冲突 |

**例 7.2**：最复杂指令 5 阶段：IF 40ns、ID 20ns、EX 15ns、MEM 40ns、WB 20ns。

1. 单周期：一条最复杂指令需 $40+20+15+40+20=135ns$。若主频 `10MHz`，周期 `100ns`，不够。
2. 多周期：周期取最长阶段 `40ns`，最大主频 `25MHz`。
3. 若 5/4/3/2 阶段指令比例为 `10%、10%、40%、40%`：
   $$CPI=5\times0.1+4\times0.1+3\times0.4+2\times0.4=2.9$$
4. 流水线理想周期也是 `40ns`，理想情况下每 `40ns` 完成一条指令。
5. 理想加速比：对单周期 $135/40=3.375$ 倍；对多周期 $2.9$ 倍。
## 基于 Yy-202 模型机或其他模型机的系统结构，进行指令执行过程的分析（已知执行过程，分析指令功能）与设计（已知指令功能，分析执行过程）  
原理：模型机题的核心是把指令功能拆成一串微操作，并对应到数据通路。

分析套路：

1. 判断指令几个字。
2. 判断操作数类型和寻址方式。
3. 写公共取指流程。
4. 写执行阶段每个机器周期的数据传送。
5. 写每步控制信号。

**例 7.3：写执行过程**

| 指令 | 功能 | 分析重点 |
| --- | --- | --- |
| `ADD2 DR,[addr]` | `Reg[DR]+Mem[addr]→Reg[DR]` | 第二字是直接地址，要再访存取操作数 |
| `JMP2 Disp` | `PC+Disp→PC` | 第二字是相对位移 |
| `MOV DR,[SR]` | `Mem[Reg[SR]]→Reg[DR]` | 有效地址由 SR 给出 |

**例 7.4：由流程图反推指令**

1. `M2~M3` 再次访问主存，说明指令有第二字，读出的 `X` 送 `DA1`。
2. `M4~M5` 计算有效地址：$EA=Reg[SR]+X$。
3. `M6` 从存储器读数据送 `DR`。
4. 指令功能：`Mem[Reg[SR]+X]→Reg[DR]`。
5. 助记符可写：`MOV DR,[SR+X]` 或 `LDA DR,[SR+X]`。
6. 源操作数是存储器类型，寻址方式是相对寄存器寻址。
## 微程序控制器的基本原理、微指令格式的设计
原理：微程序控制器用“存储逻辑”产生控制信号。每条机器指令对应一段微程序，微程序由多条微指令组成，微指令发出微命令控制硬件。

微指令常见格式：

```text
操作控制字段 + 判别测试字段 + 下址字段
```

| 控制字段编码 | 说明 |
| --- | --- |
| 直接控制法 | 每个微操作控制信号占 1 位 |
| 字段直接编译法 | 把互斥微命令分组编码，节省位数 |
| 字段间接编译法 | 进一步通过间接译码控制 |

**例 7.5：微指令字长**

题设：50 个微操作控制信号；微程序转移方式 5 种；下址字段 8 位；判别测试字段采用译码。

1. 直接控制法：控制字段 50 位；判别测试字段 $\lceil\log_2 5\rceil=3$ 位；下址字段 8 位。微指令字长 $50+3+8=61$ 位，控存容量 $256\times61$ 位。
2. 字段直接编译法：6 组控制信号分别为 `6、8、13、15、3、5` 个，编码位数为 `3+4+4+4+2+3=20` 位。微指令字长 $20+3+8=31$ 位，控存容量 $256\times31$ 位。

**例 7.6：MAPROM 容量**

操作码固定 6 位，所以最多 $2^6=64$ 条机器指令；每条平均 5 条微指令，另有取指微程序 3 条。

1. 控制存储器至少包含：$3+64\times5=323$ 个单元。
2. MAPROM 地址是 6 位操作码，共 64 项；数据是对应微程序入口地址，位数取决于控存地址位数。
# 第 8 章 RISC-V 与 ARM 模型机设计实例
## RISC-V：运算及传送指令的数据通路设计（实验 8）  
原理：RISC-V 模型机采用多周期、直接连接、哈佛结构。运算及传送指令主要包括 R 型、I 型运算和 U 型 `lui`。

**R 型指令**：

```text
Reg[rs1] op Reg[rs2] → Reg[rd]
```

| 周期 | 阶段 | 微操作 | 控制信号 |
| --- | --- | --- | --- |
| M0 | IF | `IMem[PC]→IR, PC+4→PC` | `IR_Write, PC_Write` |
| M1 | ID | `Reg[rs1]→A, Reg[rs2]→B` | 无显式写使能 |
| M2 | EX | `A op B→F/FR` | `ALU_OP` |
| M3 | WB | `F→Reg[rd]` | `Reg_Write` |

**I 型运算指令**：

```text
Reg[rs1] op imm32 → Reg[rd]
```

新增控制信号 `rs2_imm_s=1`，让 ALU 的 B 输入选择立即数 `imm32`。

**U 型 `lui`**：

```text
imm32 → Reg[rd]，imm32={imm20,12个0}
```

`lui` 只需 IF 和 WB；寄存器写入数据选择 `imm32`，对应 `w_data_s=1`。
## RISC-V：访存指令的数据通路设计（实验 9）  
原理：访存指令要先用 ALU 计算有效地址 EA，再访问数据存储器 DM。

| 指令 | 功能 | 周期 |
| --- | --- | --- |
| `lw` | `DMem[Reg[rs1]+imm32]→Reg[rd]` | IF、ID、EX 算 EA、MEM 读、WB |
| `sw` | `Reg[rs2]→DMem[Reg[rs1]+imm32]` | IF、ID、EX 算 EA、MEM 写 |

关键数据通路：

| 部件/信号 | 作用 |
| --- | --- |
| `F` 暂存器 | 保存 ALU 算出的有效地址 EA |
| DM | 数据存储器 |
| MDR | 保存 DM 读出的数据，供下一周期写回寄存器 |
| `Mem_Write` | 控制 DM 写入 |
| `w_data_s=2` | 写回寄存器时选择 MDR |

`lw` 执行过程：`取指 → 读寄存器 → A+imm32算EA → Mem32[F]→MDR → MDR→Reg[rd]`。

`sw` 执行过程：`取指 → 读寄存器 → A+imm32算EA → B→Mem32[F]`。
## RISC-V：转移类指令的数据通路设计（实验 10）  
原理：转移类指令的本质是修改 PC。模型机中 PC 来源扩展为 `PC+4`、`PC0+imm32`、`F`。

| 指令 | 功能 | 数据通路要点 |
| --- | --- | --- |
| `beq` | 相等则 `PC0+imm32→PC` | ALU 做 `A-B`，看 `ZF` |
| `jal` | `PC→rd`，`PC0+imm32→PC` | PC 已经加 4，所以取指前 PC 要保存在 `PC0` |
| `jalr` | `PC→rd`，`rs1+imm32→PC` | ALU 算跳转目标到 `F` |

新增部件/信号：

| 名称 | 作用 |
| --- | --- |
| `PC0` | 保存取指前的 PC |
| 转移地址加法器 | 计算 `PC0+imm32` |
| `PC_s` | 选择写入 PC 的来源 |
| `w_data_s=3` | `jal/jalr` 写返回地址到 rd |

`beq` 执行：`取指并保存PC0 → 读rs1/rs2 → A-B置ZF → 若ZF=1则PC0+imm32→PC`。

`jalr` 执行：`取指并保存PC0 → 读rs1 → A+imm32→F → PC→rd且F→PC`。
## RISC-V 模型机控制单元 CU 的设计（有限状态机，实验 8）
原理：CU 产生数据通路控制信号。RISC-V 属于典型 RISC，适合硬布线或有限状态机实现。

FSM 设计步骤：

1. 确定输入、输出和状态。
2. 根据指令执行流程画状态转移图。
3. 合并重复状态。
4. 状态编码。
5. 写下一状态函数和输出函数。

25 条目标指令合并成 `Idle, S1~S14` 共 15 个状态，可用 4 位二进制编码。

| 状态/路径 | 含义 |
| --- | --- |
| `S1` | `IMem[PC]→IR, PC→PC0, PC+4→PC` |
| R 型 | `S1→S2→S3→S4→S1` |
| `lw` | `S1→S2→S7→S8→S9→S1`，最长 |
| `jalr` | `S1→S2→S7→S12→S1` |
| `beq` | 若 `ZF=1`，则 `PC_Write=1, PC_s=01` |

输出信号分三类：写使能信号、多路选择器选择信号、ALU 运算码。
# 第 9 章 输入输出系统
## 外设的编址方式、I/O 接口的功能和组成  
原理：CPU 访问外设，本质上是访问外设接口中的寄存器，也叫端口。

| 编址方式 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 统一编址/存储器映射 | I/O 端口和主存共用地址空间 | 不需专用 I/O 指令，可用访存指令访问外设 | 占用主存地址空间，译码可能复杂 |
| 独立编址/I/O 映射 | I/O 端口有独立地址空间 | I/O 地址短，译码快，`IN/OUT` 可读性强 | 需要专用 I/O 指令和控制信号 |

**举例**：CPU 地址总线 16 位，前一半主存、后一半 I/O：

- `A15=0`：主存地址译码器工作。
- `A15=1`：I/O 地址译码器工作。

I/O 接口功能：

| 功能 | 说明 |
| --- | --- |
| 数据缓冲 | 协调 CPU 和外设速度差 |
| 执行命令 | 保存并译码 CPU 命令 |
| 返回状态 | CPU 读取外设是否就绪、是否出错 |
| 设备选择 | 端口地址译码选中设备 |
| 格式/信号转换 | 完成数据格式、电平、时序转换 |
| 中断管理 | 产生、屏蔽、响应中断请求 |

I/O 接口组成：命令寄存器、状态寄存器、数据缓冲寄存器、端口地址译码电路、中断控制逻辑等。
## 主机与外设交换信息的方式（四种方式）  
原理：这一节讨论 CPU 在何时、以什么方式对外设读写，目标是提高传输效率。

| 方式 | 原理 | 特点 | 适用 |
| --- | --- | --- | --- |
| 程序查询 | CPU 不断查询外设是否准备好 | 硬件简单，CPU 效率低 | 低速、少量数据 |
| 程序中断 | 外设准备好后主动请求 CPU 服务 | CPU 与外设可并行 | 随机、低中速 I/O |
| DMA | DMAC 控制主存与外设成块传输 | CPU 只初始化和收尾 | 高速、大批量数据 |
| 通道/IOP | 专门 I/O 处理机管理 I/O | 进一步减轻 CPU 负担 | 大型系统 |

**DMA 原理**：

1. 初始化阶段：CPU 设置传送方向、主存起始地址、字节数等。
2. 数据传送阶段：DMAC 申请总线，CPU 响应后，外设和主存直接交换数据。
3. 传送结束阶段：DMAC 通知 CPU，CPU 收尾或处理错误。
## 中断系统：中断概念及过程、中断分类、中断屏蔽、中断响应、中断嵌套、中断判优、中断向量
原理：中断是在 CPU 执行程序过程中，因某事件发生而暂停当前程序，转去处理事件，处理完再回到断点继续执行。

| 概念 | 含义 |
| --- | --- |
| 中断源 | 引起中断的事件或设备 |
| 硬中断/外中断 | 外设或 CPU 外部事件引起，如 I/O 请求、时钟、电源故障 |
| 软中断/内中断 | 指令或程序异常引起，如 `INT n`、除零、溢出 |
| 中断类型号 | 中断源编号，用于寻找中断服务程序入口 |
| 中断向量 | 中断服务程序入口地址 |

中断过程：

| 阶段 | 做什么 |
| --- | --- |
| 中断请求 | 中断源向 CPU 提出请求 |
| 中断响应 | CPU 执行中断隐指令，保存断点/PSW，识别中断源，转入服务入口 |
| 中断服务 | 保护现场、处理中断、恢复现场 |
| 中断返回 | 执行中断返回指令，恢复 PC/PSW，回到断点继续执行 |

**中断屏蔽**：每个中断源可设置屏蔽触发器 `INTMi`。`INTMi=1` 表示屏蔽，CPU 不响应；`INTMi=0` 表示开放。

**中断判优**：

| 方法 | 特点 |
| --- | --- |
| 软件查询 | CPU 逐个查询中断源，灵活但慢 |
| 硬件排队判优 | 硬件自动选优先级，快但优先级通常固定 |

**中断嵌套**：执行某个中断服务程序时，又响应了更高优先级中断。需要保护现场、设置屏蔽字，避免返回时现场混乱。

**8086 中断向量表**：

| 项目 | 值 |
| --- | --- |
| 中断类型数 | 256 |
| 每个向量 | 4B，`CS:IP` |
| 向量表大小 | `256×4B=1KB` |
| 地址范围 | `00000H~003FFH` |
| 类型号 N 的向量地址 | `N×4` |

**ARM 中断举例**：

1. 保存下一条指令地址到对应模式的 `LR_mode`。
2. 把 `CPSR` 复制到对应异常模式的 `SPSR`。
3. 设置 `CPSR` 相关位，切换工作模式。
4. 强制 PC 从异常向量地址取指，跳转到异常处理程序。
5. 返回时恢复寄存器、恢复 `CPSR`，把 `LR_mode` 送回 PC。
