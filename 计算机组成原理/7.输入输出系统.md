# 7.2 I/O接口
## I/O接口作用
- 数据缓冲：通过数据缓冲器（DBR）达到主机和外设工作速度的匹配
- 错误或状态监测：通过状态寄存器反馈设备的各种错误、状态信息
- 控制和定时：接收从控制总线发来的控制信号、时钟信号
- 数据格式转换：串-并、并-串等格式转换
- 与主机和设备通信：实现 主机-I/O接口-I/O设备 之间的通信
## 工作原理
![[Pasted image 20260620164303.png]]
1. **发命令**：发送**命令字**到I/O状态寄存器，向设备发送命令
2. **读命令**：从状态寄存器读取**状态字**，获得设备或I/O控制器的状态信息
3. **读/写数据**：从数据缓冲寄存器发送或读取数据，完成主机与外设的数据交换

# 7.3.2.1 中断的作用和原理
**关中断**：IF=1表示==开中断==（允许中断），IF=0表示==关中断==（不允许中断）
## 工作流程
1. 中断请求
	1. 中断源向CPU发送中断请求信号
2. 中断响应
	1. 响应中断的条件
	2. 中断判优：多个中断源同时提出请求时通过中断判优逻辑相应一个中断源
3. 中断处理
	1. 中断隐指令
	2. 中断服务程序

### 中断请求标记寄存器

### 响应中断的条件
1. 中断源由中断请求
2. CPU允许中断，即开中断
3. 一条指令执行完毕，且没有更紧迫的任务
## 中断判优-实现
**优先级设置**：
1. 硬件故障中断属于最高级，其次是软件中断
2. 非屏蔽中断优于可屏蔽中断
3. DMA请求优于I/O设备传送的中断请求
4. 高速设备优于低速设备
5. 输入设备优于输出设备
6. 实时设备优于普通设备

## 过程
1. 接受外设外出的DMA请求（外设传送一个字的请求），并向CPU发出总线请求
2. CPU响应此总线请求，发出总线响应信号，接管总线控制权，进入DMA操作周期
3. 确定传送数据的主存单元地址及长度，并能自动修改主存地址计数和传送长度计数
4. 规定数据在主存和外设间的传送方向，发出读写等控制信号，执行数据传送操作
5. 向CPU报告DMA操作的结束

## DMA方式与中断方式对比
|  项目  |           中断            |             DMA             |
| :--: | :---------------------: | :-------------------------: |
| 数据传送 | 程序控制<br>程序的切换 → 保存和恢复现场 |   硬件控制<br>CPU只需进行预处理和后处理    |
| 中断请求 |          传送数据           |             后处理             |
|  响应  |      指令执行周期结束后响应中断      | 每个机器周期结束均可，总线空闲时即可响应 DMA 请求 |
|  场景  |       CPU 控制，低速设备       |       DMA 控制器控制，高速设备        |
| 优先级  |        优先级低于 DMA        |           优先级高于中断           |
| 异常处理 |         能处理异常事件         |            仅传送数据            |
