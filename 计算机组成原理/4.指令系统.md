# 4.1.1 指令的基本格式
## 指令定义
计算机执行某种操作的命令，是计算机运行的最小功能单位。
一台计算机的所有指令的集合构成该计算机的**指令系统**，也称作**指令集**。
## 指令格式
### 操作码（OP）
用户要干什么
### 地址码（A）
对谁进行操作
## 按地址码个数分类
### 零地址指令
![[Pasted image 20260617192902.png]]
**出现的场景**：
1. 不需要操作数，如空操作、停机、关中断等指令
2. 堆栈计算机，两个操作数隐含存放在栈顶和次栈定，计算结果压回栈顶
### 一地址指令
![[Pasted image 20260617192844.png]]
**出现的场景**：
1. 只需要单操作数，如+1、-1、取反、求补等
	1. 指令含义：OP（$A_1$）->$A_1$
	   需要3次访存：取指->读$A_1$->写$A_1$
2. 需要两个操作数，单其中一个操作数隐含在某个寄存器（如隐含在ACC）
	1. 指令含义：（ACC）OP（$A_1$）->ACC
	   需要2次访问：取指->读$A_{1}$
	   注：$A_1$ 指某个主存地址，（$A_1$）表示$A_1$所指向的地址中的内容
### 二地址指令
![[Pasted image 20260617193157.png]]
**出现的场景**：
1. 常用于需要两个操作数的算术运算、逻辑运算相关指令
	1. 指令含义：$(A_{1})OP(A_{2})\to A_{1}$
	   完成一条指令需要访存4次，取指$\to$读$A_{1}\to读A_{2}\to写A_{1}$

### 三地址指令
![[Pasted image 20260617193607.png]]
**出现的场景**：
1. 常用于两个操作数的算术运算、逻辑运算相关指令
	1. 指令含义：$(A_{1}OP(A_{2}))\to A_{3}$
	   完成一条指令需要访存4次，$取指\to读A_{1}\to读A_{2}\to 写A_{3}$
### 四地址指令
![[Pasted image 20260617193847.png]]
**出现的场景**
1. 执行指令后，将PC值修改为$A_{4}$所指地址
	1. 指令含义：$(A_{1})OP(A_{2})\to A_{3}，A_{4}=下一条将要执行指令的地址$
	   完成一条指令需要访存4次，$取指\to读A_{1}\to读A_{2}\to 写A_{3}$
## 按指令长度分类
**指令字长**：一条指令的总长度（可能会变）。指令字长会影响取指令所需时间
### 定长指令字系统
指令系统中所有指令的长度都相等
### 变长指令字结构
指令系统中各种指令的长度不等
## 按操作码长度分类
### 定长操作码
指令系统中所有指令的操作码长度都相同
$n位\to2^{n}条指令$
### 可变长操作码
指令系统中各指令的操作码长度可变
## 按操作类型分类
### 数据传输
**数据传送类**：进行主存和CPU之间的数据传送

| 指令    | 源   | 目的  | 作用             |
| ----- | --- | --- | -------------- |
| LOAD  | 存储器 | 寄存器 | 把存储器中的数据放到寄存器中 |
| STORE | 寄存器 | 存储器 | 把寄存器中的数据放到存储器中 |

### 算术逻辑操作
**运算类**

| 指令  | 举例       |
| --- | -------- |
| 算术  | 加减乘除.... |
| 逻辑  | 与或非....  |
### 移位操作
**运算类**
算数移位、逻辑移位、循环移位
### 转移操作
**程序控制类**：改变程序执行的顺序

| 类型      | 具体含义                           |
| ------- | ------------------------------ |
| 无条件转移   | JMP                            |
| 条件转移    | JZ：结果为0<br>JO：结果溢出<br>JC：结果有进位 |
| 调用和返回   | CALL和RETURN                    |
| 陷阱与陷阱指令 | Trap                           |
### 输入输出操作
**输入输出类（I/O）**：进行CPU与I/O设备之间的数据传送
CPU寄存器和IO端口之间的数据传送（端口即IO接口中的寄存器）
# 4.1.4 扩展操作码指令格式
定长指令字结构+可变长操作码

| 分类    | 假定：指令字长为16位，每个地址码占4位。                            |
| ----- | ------------------------------------------------ |
| 三地址指令 | 0000~1110，共15条指令。1111留作扩展操作码使用                   |
| 二地址指令 | 1111 0000 ~ 1111 1110，共15条指令，1111 1111留作2扩展操作码使用 |
| 一地址指令 | 1111 1111 0000~ 1111 1111 1110，共15条              |
| 零地址指令 | 共16条，最后四位：0000~1111                              |
设地址长度为n，上一层留出m种状态，下一层可扩展出$m \times 2^n$种状态
## 设计原则
1. 不允许短码是长码的前缀
2. 各指令操作码不能重复

EA是真实地址，A是地址码
## 直接寻址
指令字的形式地址A就是操作数的真实地址EA，即EA=A
一条指令的执行：取指令访存一次，执行指令访存一次，一共访存两次。
**优点**：简单，指令执行阶段仅访问一次主存，不需要专门计算操作数的地址
**缺点**：A的位数决定了该指令操作数的寻址范围，操作数的地址不易修改
## 间接寻址
指令的地址字段给出的形式地址不是操作数的真正地址，而是操作数有效地址所在的存储单元的地址，也就是操作数地址的地址，即EA=(A)
![[Pasted image 20260617203529.png]]
![[Pasted image 20260617203640.png]]
**优点**：
1. 可扩大寻址范围（有效地址EA的位数大于形式地址A的位数）
2. 便于编制程序（用间接寻址可以方便地完成子程序返回）

**缺点**：
1. 指令在执行阶段需要多次访存
## 寄存器寻址
在指令字中直接给出操作数所在的寄存器编号，即$EA=R_{i}$，其操作数在由$R_{i}$所指的寄存器内。
一条指令的执行：取指令访存一次，执行指令访存0次。
![[Pasted image 20260617204148.png]]
**优点**：
1. 指令在执行阶段不访问主存，只访问寄存器
2. 指令字短且执行速度快，支持向量/矩阵运算

**缺点**：
1. 贵
## 寄存器间接寻址
寄存器$R_{i}$中给出的不是一个操作数，而是操作数所在贮存单元的地址，即$EA=(R_{i})$
![[Pasted image 20260617204326.png]]
一条指令的执行：取指令访存一次，执行指令访存一次，共访存2次

**特点**：
1. 比一般的间接寻址要快
## 隐含寻址
不是明显的给出操作数的地址，而是在指令中隐含着操作数的地址。
![[Pasted image 20260617204520.png]]
**优点**：有利于缩短指令字长
**缺点**：需增加存储操作数或隐含地址的硬件
## 立即寻址
形式地址A本身就是操作数，又称为立即数，一般采用补码形式。#表示立即寻址的特征。
![[Pasted image 20260617204736.png]]
一条指令的执行：取指令访存一次，执行指令访存0次，共访存1次
**优点**：指令执行阶段不访问主存，指令执行时间最短
**缺点**：
1. A的位数限制了立即数的范围
## 偏移寻址
以某个地址作为起点，形式地址视为“偏移量”
### 基址寻址
以程序的起始存放地址作为“起点”。
将CPU中==基址寄存器（BR）==的内容加上指令格式中的形式地址A，而形成操作数的有效地址，即==EA=(BR)+A==。
	注：基址寄存器是==面向操作系统==的，其==内容由操作系统或管理程序确定==。在程序执行过程中，基址寄存器的内容不变（作为基地址），形式地址可变（作为偏移量）
	当采用通用寄存器作为基址寄存器时，可由==用户决定哪个寄存器作为基址寄存器==，但其==内容仍由操作系统确定==。
![[Pasted image 20260617211104.png]]
![[Pasted image 20260617211243.png]]
**优点**：
1. 便于程序“浮动”，方便实现**多道程序并发运行**。
2. 可以扩大寻址范围
3. 可用于**编制浮动程序**。
### 变址寻址
程序员自己决定从哪里作为“起点”
EA=(IX)+A
![[Pasted image 20260617212059.png]]
与基址寻址的区别：变址寄存器是==面向用户==的，在程序执行过程中，==变址寄存器的内容可由用户改变==（IX作为偏移量），形式地址A不变（作为基地址）。
**优点**：适合编制循环程序
### 相对寻址
以程序计数器PC所指地址作为“起点”
EA=(PC)+A
其中**A是相对于PC**所指地址的**位移量**
![[Pasted image 20260617213135.png]]
![[Pasted image 20260617213401.png]]
**优点**：便于**程序浮动**（一段代码在程序内部的浮动），相当于寻址广泛用于转移程序
## 堆栈寻址
操作数存放在堆栈中，隐含使用堆栈指针（SP）作为操作数地址
![[Pasted image 20260617214301.png]]
### 硬堆栈
用一个专门的寄存器实现
### 软堆栈
利用主存实现

## 例子

| 指令 | 含义 | 备注 |
| --- | --- | --- |
| `mov eax, dword ptr [ebx]` | 将 `ebx` 所指主存地址的 32bit 内容复制到 `eax` 寄存器中 | `dword ptr` 表示读写 32bit |
| `mov dword ptr [ebx], eax` | 将 `eax` 的内容复制到 `ebx` 所指主存地址的 32bit 空间中 | 目的操作数在主存 |
| `mov eax, byte ptr [ebx]` | 将 `ebx` 所指主存地址的 8bit 内容复制到 `eax` 寄存器中 | `byte ptr` 表示读写 8bit |
| `mov eax, [ebx]` | 将 `ebx` 所指主存地址的内容复制到 `eax` 寄存器中 | 未指明主存读写长度时，默认 32bit |
| `mov [af996h], eax` | 将 `eax` 的内容复制到 `af996h` 所指的主存地址中 | 未指明长度时，默认 32bit |
| `mov eax, dword ptr [ebx+8]` | 将 `ebx + 8` 所指主存地址的 32bit 内容复制到 `eax` 寄存器中 | 基址加偏移 |
| `mov eax, dword ptr [af996-12h]` | 将 `af996h - 12h` 所指主存地址的 32bit 内容复制到 `eax` 寄存器中 | 地址表达式 |

# 4.3.1.3 AT&T格式 和 Intel格式
重点关注Intel格式即可。

| 对比项 | AT&T 格式 | Intel 格式 |
| :--- | :--- | :--- |
| 目的操作数 d、源操作数 s | `op s, d`<br>注：源操作数在左，目的操作数在右 | `op d, s`<br>注：源操作数在右，目的操作数在左 |
| 寄存器的表示 | `mov %ebx, %eax`<br>注：寄存器名之前必须加 `%` | `mov eax, ebx`<br>注：直接写寄存器名即可 |
| 立即数的表示 | `mov $985, %eax`<br>注：立即数之前必须加 `$` | `mov eax, 985`<br>注：直接写数字即可 |
| 主存地址的表示 | `mov %eax, (af996h)`<br>注：用小括号 | `mov [af996h], eax`<br>注：用中括号 |
| 读写长度的表示 | `movb $5, (af996h)`<br>`movw $5, (af996h)`<br>`movl $5, (af996h)`<br>`addb $4, (af996h)`<br>注：指令后加 `b`、`w`、`l` 分别表示读写长度为 byte、word、dword | `mov byte ptr [af996h], 5`<br>`mov word ptr [af996h], 5`<br>`mov dword ptr [af996h], 5`<br>`add byte ptr [af996h], 4`<br>注：在主存地址前说明读写长度 byte、word、dword |
| 主存地址偏移量的表示 | `movl -8(%ebx), %eax`<br>注：偏移量(基址)<br><br>`movl 4(%ebx, %ecx, 32), %eax`<br>注：偏移量(基址, 变址, 比例因子) | `mov eax, [ebx - 8]`<br>注：`[基址 + 偏移量]`<br><br>`mov eax, [ebx + ecx*32 + 4]`<br>注：`[基址 + 变址*比例因子 + 偏移量]` |

# 4.4 CISC和RISC

| 对比项目     | CISC               | RISC               |
| -------- | ------------------ | ------------------ |
| 指令系统     | 复杂、庞大              | 简单、精简              |
| 指令数目     | 一般大于 200 条         | 一般小于 100 条         |
| 指令字长     | 不固定                | 定长                 |
| 可访存指令    | 不加限制               | 只有 Load/Store 指令   |
| 各种指令执行时间 | 相差较大               | 绝大多数在一个周期内完成       |
| 各种指令使用频度 | 相差很大               | 都比较常用              |
| 通用寄存器数量  | 较少                 | 多                  |
| 目标代码     | 难以用优化编译生成高效的目标代码程序 | 采用优化的编译程序，生成代码较为高效 |
| 控制方式     | 绝大多数为微程序控制         | 绝大多数为组合逻辑控制        |
| 指令流水线    | 可以通过一定方式实现         | 必须实现               |

## CISC（Complex Instruction Set Computer）
**设计思路**：一条指令完成一个复杂的基本功能
**典型代表**：x86架构，主要用于笔记本、台式机等
## RISC（Reduced Instruction Set Computer）
**设计思路**：一条指令完成一个基本“动作”，多条指令组合完成一个复杂的基本功能
**典型代表**：ARM架构，主要用于手机、平板等
