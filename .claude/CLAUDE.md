# .claude instructions

- 本目录用于 Claude Code 相关配置与说明。
- 修改 `.claude/` 下内容时保持简洁，避免无关配置变更。

## 图片转笔记与结构化风格

- 依据图片内容整理笔记时，不做机械 OCR；要按当前笔记上下文整理成清晰、可复习的 Markdown。
- 优先沿用当前笔记的标题层级、编号列表、加粗重点、公式写法、SQL 代码块和考试速通风格。
- 对“名称/符号/命令/类型/约束 + 含义/说明”这类枚举内容，优先整理成 Markdown 表格，避免长串编号列表。
- **Good case：常见数据类型** 这类内容用表格表示：

  | 数据类型 | 含义 |
  | --- | --- |
  | `CHAR(n)` | 长度为 `n` 的字符型 |
  | `VARCHAR(n)` | 最大长度为 `n` 的变长字符型 |
  | `INT` | 长整型，通常占 4B |

- 对例题、推导、计算步骤，继续使用编号步骤；对所有代码、配置、图表语法使用对应语言标识的 fenced code block，例如 `sql`、`python`、`powershell`、`json`、`yaml`、`mermaid`；无法确定语言时使用普通代码块；公式使用 LaTeX。
- 如果图片内容只露出一部分，先记录可见内容，并在回复中说明后续图片继续补全。
