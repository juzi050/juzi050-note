## 1.3 冯·诺依曼体系结构特点

![[assets/images/Pasted image 20260625210348.png]]

1. 采用二进制表示数据和指令。
2. 指令由操作码和地址码组成。
3. 采用**存储程序控制**：程序和数据预先存入主存，执行时自动取指、译码、执行。
4. 指令通常顺序执行，分支由转移指令实现。
5. 硬件由五大部件组成。
6. 早期结构以运算器为中心；现代计算机更常按**以存储器为中心**理解。

## 1.4 计算机三种语言

## 1.5 计算机性能指标计算（重点）

| 指标 | 公式/说明 |
| --- | --- |
| 时钟周期 | $T=1/f$ |
| CPU 执行时间 | $T_{CPU}=I\times CPI\times T=\dfrac{I\times CPI}{f}$ |
| 加权平均 CPI | $\overline{CPI}=\sum w_i\times CPI_i$ |
| MIPS | $MIPS=\dfrac{f}{\overline{CPI}\times 10^6}$ |
| 存储容量 | 存储单元个数 $\times$ 存储字长 |
| 主存地址线 | $n$ 根地址线可寻址 $2^n$ 个单元 |

### PPT 例 1.1：CPI、MIPS 与程序执行时间

题设：主频 `1GHz`，4 类指令：

| 指令类型 | CPI | 出现比例 |
| --- | ---: | ---: |
| 运算及寄存器传送 | 1 | 55% |
| 分支 | 4 | 20% |
| 访存命中 Cache | 3 | 18% |
| 访存未命中 Cache | 10 | 7% |

1. 平均 CPI：
   $$\overline{CPI}=1\times0.55+4\times0.2+3\times0.18+10\times0.07=2.59$$
2. 主频 `1GHz`，时钟周期 $T=1ns$，平均每条指令时间 $2.59ns$。
3. 运算速度：
   $$MIPS=\frac{1}{2.59ns\times10^6}\approx386$$
4. 若程序 4 类指令条数分别为 `20,1,4,1`：
   $$T=(20\times1+1\times4+4\times3+1\times10)\times1ns=46ns$$
5. 若新架构主频升到 `1.5GHz`，但每类 CPI 都加 1：
   - 加权 CPI：$3.59$
   - 平均指令时间：$3.59/1.5\approx2.39ns$
   - MIPS：约 `418`，从 MIPS 看更好。
   - 但对上面那段程序：
     $$T=(20\times2+1\times5+4\times4+1\times11)\times\frac{1}{1.5}ns=48ns$$
     反而比 `46ns` 慢。

**考试结论**：评价性能不能只看主频或 MIPS，要看具体程序的指令数、CPI 和时钟周期；优化高频事件收益最大。

### PPT 例 1.2：不同指令集机器的性能比较

| 指令类型 | A | B | C | D |
| --- | ---: | ---: | ---: | ---: |
| M1 的 CPI | 1 | 3 | 4 | 3 |
| M1 指令条数/百万 | 8 | 4 | 2 | 4 |
| M2 的 CPI | 1 | 2 | 4 | 3 |
| M2 指令条数/百万 | 10 | 8 | 2 | 4 |

1. 平均 CPI：
   - $CPI_{M1}=20/9\approx2.22$
   - $CPI_{M2}=23/12\approx1.92$，只看 CPI，M2 更好。
2. 若主频相同：
   - $T_{M1}\propto 8\times1+4\times3+2\times4+4\times3=40$
   - $T_{M2}\propto 10\times1+8\times2+2\times4+4\times3=46$
   - $T_{M1}/T_{M2}=40/46=0.87$，实际 M1 更快。
3. 若 $f_{M1}=1.2GHz$ 且要求两者时间相同：
   $$40t_{M1}=46t_{M2}\Rightarrow 46f_{M1}=40f_{M2}\Rightarrow f_{M2}\approx1.38GHz$$

# 第 2 章 总线

## 2.1 总线基本概念

**总线**：信息传输的公共通道，核心特性是**分时**和**共享**。

| 特性 | 含义 |
| --- | --- |
| 电气特性 | 信号方向、有效电平范围 |
| 机械特性 | 连线类型、数量、接口形态 |
| 功能特性 | 每根线传输地址、数据还是控制信息 |
| 时间特性 | 信号有效时刻及信号间时序关系 |

## 2.2 总线分类

| 分类标准 | 类型 | 说明 |
| --- | --- | --- |
| 按连接部件 | 片内总线 | CPU 内部部件之间，短、快 |
|  | 系统总线 | CPU、主存、I/O 接口之间，含 AB/DB/CB |
|  | 通信总线 | 计算机与外设/计算机之间，如 USB、以太网、SPI |
| 按传送方式 | 并行总线 | 多位同时传输，近距离快，但线多、干扰大 |
|  | 串行总线 | 按位传输，线少，适合高速远距离 |
| 按通信时钟 | 同步总线 | 统一时钟，快但受慢设备和时钟偏斜限制 |
|  | 异步总线 | 请求/应答握手，能适配不同速度设备 |

## 2.3 总线性能指标

| 指标 | 含义/公式 |
| --- | --- |
| 总线宽度 | 一次总线操作能传输的数据位数 |
| 总线周期 | 一次总线操作所需时间 |
| 总线频率 | 单位时间总线操作次数 |
| 总线带宽 | $\text{带宽}=\dfrac{\text{总线宽度}}{8}\times\text{总线频率}$ |
| GT/s | 每秒传输次数，常用于高速串行总线 |

## 2.4 总线结构、仲裁与定时

| 内容 | 速通记法 |
| --- | --- |
| 单总线结构 | 所有部件共用一组总线，简单但容易成为瓶颈 |
| 多总线结构 | 高速和低速设备隔离，提高整体效率 |
| 总线仲裁 | 多个主设备请求总线时，决定谁先使用 |
| 集中式仲裁 | 总线控制器集中裁决，如链式查询、计数器定时查询 |
| 分布式仲裁 | 每个主设备都有仲裁逻辑 |
| 同步定时 | 公共时钟控制，速度快，适合速度接近的设备 |
| 异步定时 | 请求/应答握手，可靠性可高但速度较慢 |
| 分离式定时 | 把一个总线周期拆成请求和响应两个子周期，提高利用率 |

### PPT 例 2.1：突发传输时间

题设：同步总线 `100MHz`，总线宽度 `32位`，地址和数据复用，读 `128B`。

PPT 的解题口径：

1. 时钟周期：$T=1/100MHz=10ns$。
2. 不支持突发：每次数据传输都要“地址周期 + 数据周期”，按 PPT 得
   $$t_1=2T\times128=2560ns=2.56\mu s$$
3. 支持突发：只给一次首地址，后续连续传数据：
   $$t_2=T+128T=1290ns=1.29\mu s$$

**注意**：若题目明确“32 位总线一次传 4B”，严格计算应把 `128B` 换成 `32` 次 32 位传输；考试时按题干/老师口径处理。

| 转换 | 方法 |
| --- | --- |
| 任意进制 → 十进制 | 按位权加权求和 |
| 十进制整数 → 二进制 | 除 2 取余，逆序写 |

| 二进制 ↔ 八进制 | 小数点为界，3 位一组 |
| 二进制 ↔ 十六进制 | 小数点为界，4 位一组 |

## 3.2 机器数与真值

| 概念 | 含义 |
| --- | --- |
| 真值 | 数在数学上的真实值，如 `-5` |
| 机器数 | 真值在机器中的编码形式 |
| 定点数 | 小数点位置固定，分定点整数和定点小数 |
| 浮点数 | 小数点位置由阶码决定，形如 $M\times 2^E$ |

## 3.3 原码、反码、补码、移码

设机器字长为 `n+1` 位，其中 1 位符号位、`n` 位数值位。

| 编码 | 正数 | 负数 | 0 | 范围（整数） | 常用场景 |
| --- | --- | --- | --- | --- | --- |
| 原码 | 符号 0，数值不变 | 符号 1，数值不变 | +0/-0 | $-(2^n-1)$ 到 $+(2^n-1)$ | 直观、浮点尾数符号 |
| 反码 | 同原码 | 符号位不变，数值位取反 | +0/-0 | 同原码 | 过渡理解 |
| 补码 | 同原码 | 反码末位 +1 | 唯一 0 | $-2^n$ 到 $2^n-1$ | 整数运算核心 |
| 移码 | 补码符号位取反 | 补码符号位取反 | 唯一 0 | 同补码 | 浮点阶码 |

**速记**：

- 补码负数：`按位取反 + 1`。
- 移码 = 补码的符号位取反。
- 计算机中的 `int` 通常用补码表示。

### PPT 例：定点机器数

1. 原码：
   - $X=+0.1001$，$[X]_原=0.1001$
   - $X=-0.1001$，$[X]_原=1.1001$
   - $X=+1110$，$[X]_原=0,1110$
   - $X=-1110$，$[X]_原=1,1110$
2. 移码：
   - $X=1110$，$[X]_移=1,1110$
   - $X=-1110$，$[X]_移=0,0010$
3. 已知机器数求真值：
   - $[X]_补=1,0000000$（8 位）表示 $-128$。
   - $[X]_原=1,0000000$ 表示 $-0$。
   - $[X]_移=1,0000000$ 表示 $0$。

## 3.4 浮点数规格化表示

浮点数一般写为：

$$N=M\times 2^E$$

| 部分 | 含义 | 常见编码 |
| --- | --- | --- |
| 阶码 $E$ | 指数，决定小数点位置 | 移码 |
| 尾数 $M$ | 有效数字，决定精度 | 原码或补码 |

规格化判断：

| 尾数编码 | 规格化条件 |
| --- | --- |
| 原码尾数 | 最高有效位为 1，即 $M_1=1$ |
| 补码尾数 | $M_s\oplus M_1=1$，即正数形如 `0.1...`，负数形如 `1.0...` |

### PPT 例 3.8：自定义浮点格式

格式：16 位，阶码 6 位（含阶符）用移码，尾数 10 位（含数符）用补码，阶码在前。

1. $X=-27/128=(-0.0011011)_2$
2. 规格化：$X=-0.11011\times2^{-2}$
3. 阶码和尾数编码：
   - $E_X=-00010$，$[E_X]_移=0,11110$
   - $M_X=-0.110110000$，$[M_X]_补=1.001010000$
4. 拼接：
   $$[X]_浮=0,11110\ 1.001010000=7A50H$$

若 $[Y]_浮=99BEH$：

1. 二进制：`1001 1001 1011 1110`
2. 拆分：$[E_Y]_移=1,00110$，$[M_Y]_补=0.110111110$
3. 真值：$E_Y=+00110$，$M_Y=+0.110111110$
4. 所以：
   $$Y=0.110111110\times2^6=110111.11_2=55.75$$

## 3.5 IEEE 754 单精度浮点数

| 字段 | 位数 | 含义 |
| --- | ---: | --- |
| 数符 S | 1 | 0 正，1 负 |
| 阶码 E | 8 | 偏置移码，Bias=127 |
| 尾数 fraction | 23 | 隐藏最高位 `1`，实际尾数为 `1.fraction` |

规格化数真值：

$$(-1)^S\times(1.fraction)_2\times2^{E-127}$$

### PPT 例 3.10：IEEE754 编码与解码

1. $X=-135.625$
   - 二进制：$-10000111.101_2$
   - 规格化：$-1.0000111101\times2^7$
   - 阶码：$127+7=134=10000110_2$
   - 数符：`1`
   - 尾数：`00001111010000000000000`
2. $Y=41360000H$
   - 二进制拆分：`0 10000010 01101100000000000000000`
   - $S=0$，$E=130$，真阶码 $E'=3$
   - 真值：$+1.011011_2\times2^3=11.375$

### PPT 练习：为什么 `float x=0.1` 不精确

`0.1` 的单精度编码对应：

$$0.100000001490116119384765625$$

所以它比数学上的 `0.1` 略大。**考点**：浮点数表示有限，很多十进制小数无法精确表示。

## 3.6 数据类型转换与大小端

| 转换 | 规则 |
| --- | --- |
| 有符号短整数 → 长整数 | 符号扩展 |
| 无符号短整数 → 长整数 | 零扩展 |
| 长整数 → 短整数 | 截断低位，可能溢出 |
| 浮点 ↔ 整型 | 可能产生舍入、溢出、精度损失 |
| 小端模式 | 低有效字节放低地址 |
| 大端模式 | 高有效字节放低地址 |

### PPT 例 3.12：指针、小端与补码

32 位机器，小端模式：

```c
int *p;
int x = -123456;
p = &x;
int y = *p;
```

分析套路：

1. `x` 是 32 位补码。
2. `p=&x`，指针变量保存 `x` 的首地址。
3. `y=*p`，按 `int` 取 4B，得到与 `x` 相同的值。
4. 小端只影响内存中字节顺序，不改变寄存器中数值含义。

### PPT 例 3.13：强制类型转换

| 语句 | 十进制值 | 十六进制编码 | 说明 |
| --- | ---: | --- | --- |
| `short si=-1234;` | -1234 | `FB2E` | 16 位补码 |
| `int i=si;` | -1234 | `FFFF FB2E` | 符号扩展 |
| `unsigned short usi=si;` | 64302 | `FB2E` | 位模式不变，解释方式变 |

# 第 4 章 运算方法与运算器

## 4.1 运算操作与移位

| 运算类型 | 说明 |
| --- | --- |
| 算术运算 | 加、减、乘、除，分有符号/无符号、整数/浮点 |
| 按位运算 | 与、或、非、异或 |
| 移位运算 | 左移相当于乘 2，右移相当于除 2 |
| 逻辑运算 | 真/假判断，通常由条件码和转移实现 |

| 移位 | 对象 | 空位补什么 |
| --- | --- | --- |
| 逻辑左移 | 无符号/位串 | 低位补 0 |
| 逻辑右移 | 无符号/位串 | 高位补 0 |
| 算术左移 | 补码有符号数 | 低位补 0，可能溢出 |
| 算术右移 | 补码有符号数 | 高位补符号位 |
| 循环移位 | 位串 | 移出位回填 |

**补码算术左移判溢**：移位前若符号位和最高数值位不同，则左移后会溢出。

## 4.2 补码加减运算

核心公式：

$$[X+Y]_补=[X]_补+[Y]_补$$

$$[X-Y]_补=[X]_补+[-Y]_补$$

补码好处：符号位和数值位一起参加运算，减法可转化为加法。

### 溢出判断

| 方法 | 判断 |
| --- | --- |
| 单符号位 | 同号相加结果异号；正数减负数结果为负；负数减正数结果为正 |
| 进位判溢 | 最高数值位进位 $C_1$ 与符号位进位 $C_s$ 不同，则溢出，$V=C_1\oplus C_s$ |
| 双符号位 | 结果两个符号位不同则溢出；`01` 正溢出，`10` 负溢出 |

### PPT 例 4.2：补码加减

已知 $X=+1011$，$Y=-0100$：

1. 写补码：
   - $[X]_补=0,1011$
   - $[Y]_补=1,1100$
   - $[-Y]_补=0,0100$
2. 计算：
   - $[X+Y]_补=0,0111$，即 $+7$
   - $[X-Y]_补=0,1111$，即 $+15$

### PPT 例 4.4：判溢

1. $X=+1000$，$Y=+1001$，算 $X+Y$：
   - 正数 + 正数却得到负数，正溢出。
   - 进位法：$C_1\neq C_s$，溢出。
2. $X=+1000$，$Y=+1001$，算 $X-Y$：
   - 等价于 $X+[-Y]$。
   - 若 $C_1=C_s$，无溢出。

## 4.3 移码加减

移码常用于浮点阶码。

| 运算 | 做法 |
| --- | --- |
| $X+Y$ | $[X]_移+[Y]_补$，结果仍为移码 |
| $X-Y$ | $[X]_移+[-Y]_补$，结果仍为移码 |
| 阶码正溢出 | 浮点数上溢 |
| 阶码负溢出 | 浮点数下溢，常按机器零处理 |

### PPT 例 4.5

$X=+1000$，$Y=+1001$：

- $[X]_移=01,1000$
- $[Y]_补=00,1001$
- $[-Y]_补=11,0111$

计算：

- $[X+Y]_移=01,1000+00,1001=10,0001$，正溢出。
- $[X-Y]_移=01,1000+11,0111=00,1111$，无溢出，$X-Y=-1$。

## 4.4 乘除法算法速通

| 运算 | PPT 算法 | 关键词 |
| --- | --- | --- |
| 原码一位乘法 | 累加、右移 | 符号单独处理，绝对值相乘 |
| 补码 Booth 乘法 | 根据 $Y_{n+1}-Y_n$ 决定位积 | `+X`、`-X`、`+0`，部分积和乘数联合右移 |
| 原码恢复余数除法 | 减除数，若余数负则恢复 | 减/恢复余数、左移 |
| 原码不恢复余数除法 | 加减交替 | 余数正则减，余数负则加 |

### PPT 例 4.6：原码一位乘法

$X=+1011$，$Y=-1101$，求 $P=X\cdot Y$。

1. $[X]_原=0,1011$，$[Y]_原=1,1101$。
2. 积符号：$P_s=X_s\oplus Y_s=1$。
3. 绝对值相乘：$|P|=1011\times1101=10001111$。
4. 结果：$[P]_原=1,10001111$。

## 4.5 运算器结构与数据通路

| 结构 | 特点 |
| --- | --- |
| 单总线运算器 | 结构简单，但同一周期总线上只能有一个数据 |
| 双总线/多总线 | 可并行读多个操作数，周期更少 |
| 直接连接 | 专用线路连接，复杂但效率高，适合流水线 CPU |

### PPT 例 4.10：单总线执行 `ADD R3,R2,R7`

功能：`R2 + R7 → R3`

| 周期 | 操作 | 控制信号示意 |
| --- | --- | --- |
| 1 | `R2 → LA` | `RA=2, GRout=1, LAin=1` |
| 2 | `R7 → LB` | `RA=7, GRout=1, LBin=1` |
| 3 | `ALU: A+B → R3` | `ALU_OP=加法, ALUout=1, GRin=1, WA=3` |

结论：需要 3 个 CPU 周期。

### PPT 例 4.11：双总线执行 `ADD R3,R2,R7`

| 周期 | 操作 |
| --- | --- |
| 1 | `R2→IB1, R7→IB2, F=A+B, F→LC` |
| 2 | `LC→R3` |

结论：需要 2 个 CPU 周期。

## 4.6 标志寄存器

| 标志 | 含义 | 常见判断 |
| --- | --- | --- |
| ZF | 零标志 | 结果为 0 则 1 |
| SF | 符号标志 | 结果最高位为 1 则 1 |
| CF | 进位/借位标志 | 无符号加法进位或减法借位 |
| OF | 溢出标志 | 有符号运算溢出 |
| PF | 奇偶标志 | 结果低字节 1 的个数为偶数则 1 |

### PPT 例 4.12：`80H + 80H`

```asm
MOV AL,80H
ADD AL,80H
```

结果：`AL=00H`。

| 标志 | 值 | 原因 |
| --- | ---: | --- |
| ZF | 1 | 结果为 0 |
| CF | 1 | 无符号加法产生进位 |
| OF | 1 | `-128 + -128` 超出有符号 8 位范围 |
| SF | 0 | 结果最高位为 0 |
| PF | 1 | `00H` 中 1 的个数为偶数 |

## 4.7 浮点加减运算

流程：

1. 对阶：小阶向大阶看齐，尾数右移。
2. 尾数加减。
3. 规格化：左规或右规，并调整阶码。
4. 舍入：如 0 舍 1 入。
5. 判溢：阶码上溢/下溢。

### PPT 例 4.14：浮点加法套路

格式：12 位浮点数，阶码 4 位（含阶符）、尾数 8 位（含数符），均用补码。

$$X=-0.1001011\times2^{001},\quad Y=0.1100101\times2^{-010}$$

1. 写出浮点编码：
   - $[X]_浮=0,001\ 1.0110101$
   - $[Y]_浮=1,110\ 0.1100101$
2. 对阶：$\Delta E=E_X-E_Y=3$，所以 $Y$ 的尾数右移 3 位。
3. 尾数相加后规格化、舍入、判溢。

**考试重点**：对阶一定是“小阶向大阶看齐”，右移会损失低位，需按题目要求舍入。

## 5.4 主存扩展设计（SRAM/ROM）

### 原理

| 扩展方式 | 解决问题 | 做法 |
| --- | --- | --- |
| 位扩展 | 字长不够 | 多片芯片地址线并联，数据线拼宽，片选相同 |
| 字扩展 | 字数不够 | 数据线并联，高位地址译码产生片选 |
| 字位扩展 | 字数和位数都不够 | 先位扩展成一组，再用多组做字扩展 |

### 与 CPU 连接步骤

1. 根据 CPU 地址线确定可寻址范围。
2. 根据目标地址空间确定芯片容量和片数。
3. 低位地址线接芯片片内地址。
4. 高位地址线接译码器，产生片选信号。
5. 数据线按位宽连接。
6. 读写控制线连接 SRAM/RAM；ROM 不接写信号。
7. `MREQ#` 常接译码器使能端。

### PPT 例：Cache 地址划分

题设：主存 `4GB`，按字节编址，块大小 `16B`，访问地址 `234589ABH`。

| 映射 | Cache 条件 | 块内地址 | 行/组地址 | 标记 |
| --- | --- | ---: | ---: | ---: |
| 直接映射 | Cache `512KB` | 4 位：`BH` | 15 位：`589AH` | 13 位：`468H` |
| 全相联 | Cache `8KB` | 4 位：`BH` | 无 | 28 位：`234589AH` |
| 4 路组相联 | Cache `512KB` | 4 位：`BH` | 13 位：`189AH` | 15 位：`11A2H` |

计算套路：

1. 块大小 `16B=2^4B`，所以块内地址 4 位。
2. Cache 行数 = Cache 容量 / 块大小。
3. 直接映射：行号位数 = $\log_2 行数$。
4. 组相联：组数 = 行数 / 路数，组号位数 = $\log_2 组数$。
5. 标记位数 = 主存地址位数 - 行/组号位数 - 块内地址位数。

## 6.1 指令系统基本概念

| 概念 | 含义 |
| --- | --- |
| 指令系统 | 一台计算机全部机器指令的集合 |
| ISA | 指令集体系结构，是软硬件接口 |

机器指令通常由：

```text

```

| 字段 | 作用 |
| --- | --- |
| 操作码 | 指明做什么操作 |
| 地址码 | 指明操作数或操作数地址 |

## 6.2 地址码格式

| 类型 | 功能形式 | 说明 |
| --- | --- | --- |
| 三地址 | `(A1) OP (A2) → A3` | 两源一目的 |
| 二地址 | `(A1) OP (A2) → A1` | 目的操作数兼源操作数 |
| 一地址 | `(ACC) OP (A) → ACC` | 一个操作数隐含在 ACC |
| 零地址 | 无显式地址 | 如 `NOP/HLT` 或堆栈指令 |

## 6.3 操作码扩展技术

**原理**：把不用的地址码字段扩展为操作码，使不同地址数的指令有不同长度的操作码。

| 优点 | 缺点 |
| --- | --- |
| 缩短平均指令长度；增加可表示操作数 | 指令译码复杂，控制器设计难度增加 |

## 6.4 寻址方式（重点）

| 寻址方式 | 有效地址/操作数 | 速记 |
| --- | --- | --- |
| 立即寻址 | `Operand=A` | 操作数就在指令中 |
| 直接寻址 | `EA=A`，`Operand=Mem[A]` | 地址在指令中 |
| 间接寻址 | `EA=Mem[A]`，`Operand=Mem[EA]` | 地址的地址 |
| 寄存器寻址 | `Operand=Reg[Ri]` | 操作数在寄存器 |
| 寄存器间接 | `EA=Reg[Ri]`，`Operand=Mem[EA]` | 地址在寄存器 |
| 变址寻址 | `EA=RI+A` | 数组/字符串，下标变化 |
| 基址寻址 | `EA=Rb+A` | 程序重定位/扩大寻址范围 |
| 相对寻址 | `EA=PC+A` | 转移指令常用 |
| 堆栈寻址 | `EA=SP` | 中断/子程序现场与返回地址 |

### PPT 例 6.3：变址寻址判断

x86 程序用变址寄存器遍历数组 `num` 的 10 个 16 位数据：

1. 功能：把 `num` 数组 10 个数据累加，和放在 `AX`。
2. 每次变址寄存器 `ESI += 2`，说明一个 16 位数据占 2 个地址。
3. 结论：x86 按**字节编址**。
4. 对应 C 语言就是固定次数 `for` 循环累加数组。

## 6.7 PPT 例题

### PPT 例 6.5：C 到 RV32I

```c
int a,b,c,w;
a = -3;
b = 1000;
c = 7000;
w = 8*a + b - c;
```

假设 `a,b,c,w` 分配到 `x5~x8`：

```asm
addi x5,x0,-3       # a=-3
addi x6,x0,1000     # b=1000
lui  x7,2           # 处理 7000=0x1B58，低12位最高位为1，lui需进位
addi x7,x7,0xB58    # c=7000
slli x8,x5,3        # 8a
add  x8,x8,x6       # 8a+b
sub  x8,x8,x7       # 8a+b-c
```

**易错点**：`addi` 的 12 位立即数会符号扩展。`0xB58` 最高位为 1，会被当作负数，所以 `lui` 立即数要用 `imm32[31:12]+imm32[11]`。

### PPT 例 6.6：load/store 与小端

题设：地址 `401280H` 的 32 位数据为 `234589ABH`，小端模式。

```asm
lui  t0,0x401       # t0=0x401000
addi t0,t0,0x200    # t0=0x401200
lw   t1,0x80(t0)    # t1=234589ABH
lh   t2,0x80(t0)    # t2=FFFF89ABH，符号扩展
lhu  t3,0x80(t0)    # t3=000089ABH，零扩展
lh   t4,0x82(t0)    # t4=00002345H
lb   t5,0x80(t0)    # t5=FFFFFFABH
lb   t6,0x83(t0)    # t6=00000023H
sw   t4,0x80(t0)
sh   t3,0x82(t0)
sb   t5,0x83(t0)
```

最终 PPT 结果：主存 `401280H` 开始的 4B 内容为 `ABAB2345H`。

### PPT 例 6.9：while 循环翻译

C 语言：

```c
while (data[i] != y)
    i += 1;
```

寄存器约定：`data` 首地址在 `s4(x20)`，`i` 在 `s9(x25)`，`y` 在 `s10(x26)`。

```asm
Loop:
    slli t1,s9,2       # t1=i*4
    add  t1,t1,s4      # t1=&data[i]
    lw   t2,0(t1)      # t2=data[i]
    beq  t2,s10,exit   # 若 data[i]==y，退出
    addi s9,s9,1       # i++
    jal  x0,Loop       # 无条件回到循环入口
exit:
```

**考点**：数组地址 = 基址 + 元素大小 × 下标；条件分支用偏移量；`jal x0,label` 可当无条件跳转。

| 时序系统 | 产生机器周期、节拍、工作脉冲 |
| 控制单元 CU | 产生微操作控制信号序列 |

控制器功能：取指令、分析指令、执行指令、中断处理、响应特殊请求。

## 7.2 时序系统

| 概念 | 含义 |
| --- | --- |
| 指令周期 | 取出并执行完一条指令的时间 |
| 机器周期/CPU 周期 | 指令周期中的一个阶段 |
| 时钟周期/节拍 | CPU 主频脉冲周期 |
| 工作脉冲 | 在节拍内真正触发寄存器写入等动作 |

### PPT 例 7.1：CPI 和等待节拍

题设：主频 `100MHz`，平均每条指令 3 个机器周期，每个机器周期 2 个节拍。

1. $CPI=3\times2=6$。
2. 时钟周期 $T=10ns$。
3. MIPS：$100MHz/6\approx16.7MIPS$。
4. 若每条指令平均访存 1.5 次，每次插入 1 个等待节拍，则平均多 `1.5` 个节拍：
   $$CPI'=6+1.5=7.5$$
   $$T_{inst}=7.5\times10ns=75ns$$

## 7.3 单周期、多周期、流水线

| CPU | 特点 | 优缺点 |
| --- | --- | --- |
| 单周期 | 每条指令一个时钟周期完成 | 控制简单，但周期必须按最慢指令设计 |
| 多周期 | 一条指令分多个周期完成 | 周期可按最长阶段设计，CPI 不同 |
| 流水线 | 多条指令重叠执行 | 吞吐率高，但有相关/冲突处理 |

### PPT 例 7.2：三种实现比较

最复杂指令 5 阶段：IF 40ns、ID 20ns、EX 15ns、MEM 40ns、WB 20ns。

1. 单周期：一条指令需 $40+20+15+40+20=135ns$。若主频 `10MHz`，周期 `100ns`，不够。
2. 多周期：每阶段一个周期，周期取最长阶段 `40ns`，最大主频 `25MHz`。
3. 若 5/4/3/2 阶段指令比例为 10%、10%、40%、40%：
   $$CPI=5\times0.1+4\times0.1+3\times0.4+2\times0.4=2.9$$
4. 流水线理想周期 `40ns`，理想每 40ns 完成一条。相对加速：
   - 对单周期：$135/40=3.375$ 倍。
   - 对多周期：$(40\times2.9)/40=2.9$ 倍。

## 7.4 硬布线控制器与微程序控制器

| 类型 | 原理 | 优点 | 缺点 | 适合 |
| --- | --- | --- | --- | --- |
| 硬布线控制器 | 组合逻辑根据指令、时序、状态产生控制信号 | 快 | 设计复杂、修改困难 | RISC |
| 微程序控制器 | 每条机器指令由一段微程序解释执行 | 规整、易扩展 | 慢 | CISC |

## 7.5 Yy-z02 模型机指令执行分析

分析套路：

1. 判断指令几个字。
2. 判断操作数类型和寻址方式。
3. 写公共取指流程。
4. 写执行阶段每个机器周期的数据传送。
5. 写每步需要的控制信号。

### PPT 例 7.3：写执行过程

指令：

```asm
ADD2 DR,[addr]   ; Reg[DR] + Mem[addr] → Reg[DR]
JMP2 Disp        ; PC + Disp → PC
MOV  DR,[SR]     ; Mem[Reg[SR]] → Reg[DR]
```

分析要点：

| 指令 | 是否取第二字 | 第二字含义 | 执行动作 |
| --- | --- | --- | --- |
| `ADD2 DR,[addr]` | 要 | 直接地址 `addr` | 取 `Mem[addr]` 与 `Reg[DR]` 相加写回 |
| `JMP2 Disp` | 要 | 相对位移 | `PC+Disp→PC` |
| `MOV DR,[SR]` | 不一定另取地址字 | 地址由 SR 给出 | `Mem[Reg[SR]]→Reg[DR]` |

### PPT 例 7.4：由流程图反推指令

PPT 结论：

1. 执行周期 `M2~M6`。
2. `M2~M3` 再次访问主存，说明有第二字，读出的 `X` 送 `DA1`。
3. `M4~M5` 计算有效地址：$EA=Reg[SR]+X$。
4. `M6` 从存储器读数据送 `DR`。
5. 指令功能：

```asm
MOV DR,[SR+X]    ; 或 LDA DR,[SR+X]
```

源操作数是存储器类型，寻址方式是相对寄存器寻址。

## 7.6 微指令格式与控存容量

微指令通常含：

```text
操作控制字段 + 判别测试字段 + 下址字段
```

| 控制字段编码 | 含义 |
| --- | --- |
| 直接控制法 | 每个微操作控制信号占 1 位 |
| 字段直接编译法 | 互斥微命令分组编码，节省位数 |
| 字段间接编译法 | 进一步通过译码/间接字段控制 |

### PPT 例 7.5：微指令字长

题设：50 个微操作控制信号；微程序转移方式 5 种；下址字段 8 位；判别测试字段用译码。

1. 直接控制法：
   - 控制字段 50 位。
   - 判别测试字段：5 种转移需 $\lceil\log_2 5\rceil=3$ 位。
   - 下址字段 8 位。
   - 微指令字长：$50+3+8=61$ 位。
   - 控存容量：$2^8\times61=256\times61$ 位。
2. 字段直接编译法：6 组控制信号分别为 `6,8,13,15,3,5` 个。
   - 编码位数：`3+4+4+4+2+3=20` 位。
   - 微指令字长：$20+3+8=31$ 位。
   - 控存容量：$256\times31$ 位。

### PPT 例 7.6：MAPROM 容量

操作码固定 6 位，所以机器指令最多 $2^6=64$ 条。每条机器指令平均 5 条微指令，取指微程序 3 条。

1. 控存单元数：
   $$3+64\times5=323$$
2. 若用 MAPROM 由操作码映射入口地址：
   - 地址：6 位操作码，共 64 项。
   - 数据：微程序入口地址，至少能表示控存地址。
   - 若控存按 512 个单元规划，则入口地址 9 位，MAPROM 容量 $64\times9$ 位。

# 第 8 章 RISC-V 与 ARM 模型机设计实例

## 8.1 RISC-V 模型机总体设计

PPT 设计方案：

| 项目 | 设计 |
| --- | --- |
| 目标指令集 | RV32I 典型核心子集（25 条） |
| CPU 类型 | 多周期 CPU |
| 连接方式 | 直接连接，无共享总线 |
| 存储器结构 | 哈佛结构，指令存储器和数据存储器分离 |
| 基本阶段 | IF、ID、EX、MEM、WB |

主要部件：PC、IR、指令存储器、寄存器堆、立即数生成器、ALU、FR、A/B/F 暂存器、数据存储器、MDR、CU。

## 8.2 R/I/U 型运算与传送指令

### R 型指令

功能：

```text
Reg[rs1] op Reg[rs2] → Reg[rd]
```

执行过程：

| 周期 | 阶段 | 微操作 | 控制信号 |
| --- | --- | --- | --- |
| M0 | IF | `IMem[PC]→IR, PC+4→PC` | `IR_Write, PC_Write` |
| M1 | ID | `Reg[rs1]→A, Reg[rs2]→B` | 无显式写使能 |
| M2 | EX | `A op B→F/FR` | `ALU_OP` |
| M3 | WB | `F→Reg[rd]` | `Reg_Write` |

### I 型运算指令

功能：

```text
Reg[rs1] op imm32 → Reg[rd]
```

新增控制：`rs2_imm_s=1`，让 ALU 的 B 输入选择 `imm32`。

### U 型 `lui`

功能：

```text
imm32 → Reg[rd]，imm32={imm20,12个0}
```

只需：IF + WB。新增 `w_data_s`，让寄存器堆写入数据选择 `imm32`。

## 8.3 访存指令数据通路

| 指令 | 功能 | 周期 |
| --- | --- | --- |
| `lw` | `DMem[Reg[rs1]+imm32]→Reg[rd]` | IF、ID、EX 算 EA、MEM 读、WB |
| `sw` | `Reg[rs2]→DMem[Reg[rs1]+imm32]` | IF、ID、EX 算 EA、MEM 写 |

关键部件：

| 部件/信号 | 作用 |
| --- | --- |
| DM | 数据存储器 |
| MDR | 保存 DM 读出的数据，下一周期写回寄存器 |
| `Mem_Write` | 控制 DM 写 |
| `w_data_s=2` | 寄存器写入数据选择 MDR |
| `F` | 保存 ALU 算出的有效地址 EA |

## 8.4 转移类指令数据通路

| 指令 | 功能 | 数据通路要点 |
| --- | --- | --- |
| `beq` | 若相等则 `PC0+imm32→PC` | ALU 做 `A-B`，看 `ZF` |
| `jal` | `PC→rd`，`PC0+imm32→PC` | 需保存取指前 PC 到 `PC0` |
| `jalr` | `PC→rd`，`rs1+imm32→PC` | ALU 算跳转目标到 `F` |

新增部件/信号：

| 名称 | 作用 |
| --- | --- |
| `PC0` | 保存取指前的 PC，用于相对跳转 |
| 转移地址加法器 | 计算 `PC0+imm32` |
| `PC_s` | 选择 `PC+4`、`PC0+imm32`、`F` 写入 PC |
| `w_data_s=3` | `jal/jalr` 把返回地址写入 rd |

## 8.5 CU 设计

控制单元 CU 可用：

1. 硬布线控制：组合逻辑产生控制信号。
2. 有限状态机 FSM：适合 FPGA，本质是组合逻辑 + 时序逻辑。

FSM 设计步骤：

1. 确定输入、输出和状态。
2. 根据指令执行流程画状态转移图。
3. 合并重复状态，PPT 中合并为 `Idle, S1~S14` 共 15 个状态。
4. 状态编码：15 个状态可用 4 位二进制编码。
5. 写下一状态函数和输出函数。

PPT 关键状态：

| 状态 | 微操作/含义 |
| --- | --- |
| `S1` | `IMem[PC]→IR, PC→PC0, PC+4→PC` |
| R 型 | `S1→S2→S3→S4→S1` |
| `lw` | `S1→S2→S7→S8→S9→S1`，最长 |
| `jalr` | `S1→S2→S7→S12→S1` |
| `beq` | 若 `ZF=1`，`PC_Write=1, PC_s=01` |

## 8.6 ARM 模型机（了解）

PPT 小结：ARM 模型机也是多周期、直接连接；可模拟条件取指、工作模式切换、块数据传送等 ARM 特色机制。

# 第 9 章 输入输出系统

## 9.1 外设编址方式

| 编址方式 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 统一编址/存储器映射 | I/O 端口和主存共用地址空间 | 不需专用 I/O 指令；访存指令可访问外设 | 占用主存地址空间，译码可能复杂 |
| 独立编址/I/O 映射 | I/O 端口有独立地址空间 | I/O 地址短，译码快，可读性强 | 需专用 `IN/OUT` 指令和控制信号 |

### PPT 举例：16 位地址线一半主存一半 I/O

- `A15=0`：主存地址译码器工作。
- `A15=1`：I/O 地址译码器工作。

## 9.2 I/O 接口

CPU 访问外设的实质：**访问外设接口中的寄存器（端口）**。

| 接口功能 | 说明 |
| --- | --- |
| 数据缓冲 | 协调 CPU 与外设速度差 |
| 命令执行 | 保存并译码 CPU 命令 |
| 状态返回 | 让 CPU 读取外设状态 |
| 设备选择 | 通过端口地址译码选中设备 |
| 格式/信号转换 | 完成数据格式、电平、时序转换 |
| 中断管理 | 产生/屏蔽/响应中断请求 |

接口组成：命令寄存器、数据缓冲寄存器、状态寄存器、端口地址译码电路、中断控制逻辑等。

## 9.3 主机与外设交换信息方式

| 方式 | 原理 | 特点 |
| --- | --- | --- |
| 程序查询 | CPU 不断查询外设是否准备好 | 硬件简单，CPU 效率低 |
| 程序中断 | 外设准备好后主动中断 CPU | CPU 与外设可并行，提高效率 |
| DMA | DMAC 控制主存与外设成块传输 | 适合高速大批量数据 |
| 通道/IOP | 专门 I/O 处理机管理 I/O | 减轻 CPU 负担，适合大型系统 |

## 9.4 中断系统

### 基本概念

| 概念 | 含义 |
| --- | --- |
| 中断 | CPU 暂停当前程序，转去处理事件，处理后返回断点 |
| 中断源 | 引起中断的事件或设备 |
| 硬中断 | 外设/外部事件引起，如 I/O 请求、时钟、电源故障 |
| 软中断 | CPU 内部指令或异常引起，如 `INT n`、除零、溢出 |
| 中断类型号 | 给中断源的编号，用于查找中断向量 |
| 中断向量 | 中断服务程序入口地址 |

### 中断过程（重点）

| 阶段 | 做什么 |
| --- | --- |
| 中断请求 | 中断源向 CPU 提出请求；外中断通过请求引脚，内中断直接触发 |
| 中断响应 | CPU 执行中断隐指令，保存断点/PSW，识别中断源，转入服务入口 |
| 中断服务 | 保护现场、处理中断、恢复现场 |
| 中断返回 | 执行中断返回指令，恢复 PC/PSW，回到断点继续执行 |

**CPU 通常在当前指令执行完后检测中断请求**。

### 中断屏蔽与判优

| 机制 | 含义 |
| --- | --- |
| 中断屏蔽 | `INTMi=1` 屏蔽中断源 i；`INTMi=0` 开放 |
| 软件查询判优 | CPU 逐个查询中断源，灵活但慢 |
| 硬件排队判优 | 由硬件电路决定优先级，快但优先级较固定 |
| 向量中断 | 硬件提供中断类型号/向量地址，快速转入服务程序 |

### 8086 中断向量表

| 项目 | 值 |
| --- | --- |
| 中断类型数 | 256 |
| 每个向量 | 4B，`CS:IP` |
| 向量表大小 | `256×4B=1KB` |
| 地址范围 | `00000H~003FFH` |
| 类型号 N 的向量地址 | `N×4` |

## 9.5 DMA

**基本原理**：在 DMA 控制器（DMAC）控制下，把块数据从一个地址空间复制到另一个地址空间，传输时外设和主存可直接通过总线交换数据。

| 阶段 | 操作 |
| --- | --- |
| 初始化 | CPU 设置传送方向、主存起始地址、字节数等 |
| 数据传送 | DMAC 申请总线，CPU 响应后由 DMAC 控制传输 |
| 传送结束 | DMAC 通知 CPU，CPU 收尾/处理错误 |

### PPT ARM 中断举例

ARM 异常/中断响应过程：

1. 保存下一条指令地址到对应模式的 `LR_mode`。
2. 把 `CPSR` 复制到对应异常模式的 `SPSR`。
3. 设置 `CPSR` 相关位，切换工作模式。
4. 强制 PC 从异常向量地址取指，跳转到异常处理程序。
5. 返回时恢复寄存器、恢复 `CPSR`，把 `LR_mode` 送回 PC。
